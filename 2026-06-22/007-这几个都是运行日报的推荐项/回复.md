我理清了 tool-audit 的架构。以下是分析结论。

## 问题

推荐逻辑在 `tool_audit_full.py` 的 `build_prompt()` 中，把当前 MCP + Skills 列表发给 LLM，让 DeepSeek 自由推荐。没有"已拒绝"的记忆机制 → 下周 LLM 大概率再推同样的三个。

## 方案

在 prompt 中注入一个"**已评估并拒绝**"的 blocklist，让 LLM 在生成推荐时自然排除。

### 改动点

1. **新建** `ops-daemon/data/working/tool_audit_rejected.json` — 手工维护的拒绝清单，结构：
```json
{
  "dotenv-manager": {
    "type": "Skill",
    "reason": "已手动管理 .env，无需额外 skill",
    "evaluated": "2026-06-22"
  },
  "docker-assist": {
    "type": "Skill",
    "reason": "技术栈无 Docker 需求",
    "evaluated": "2026-06-22"
  },
  "git-conventional": {
    "type": "Skill",
    "reason": "已有 bash-hooks + CLAUDE.md 约束，重复",
    "evaluated": "2026-06-22"
  }
}
```

2. **改** `tool_audit_full.py` 的 `build_prompt()` — 在 prompt 末尾追加"以下项目已经评估过、不需要推荐"的章节：
```python
REJECTED_PATH = r"D:\ClaudeProjects\ops-daemon\data\working\tool_audit_rejected.json"

# 在 build_prompt() 末尾，拼接 rejected 内容
if os.path.exists(REJECTED_PATH):
    with open(REJECTED_PATH, "r", encoding="utf-8") as f:
        rejected = json.load(f)
    if rejected:
        lines = ["", "## 以下项目已评估并拒绝，请勿推荐"]
        for name, info in rejected.items():
            lines.append(f"- **{name}** ({info.get('type', '?')}): {info.get('reason', '')}")
        prompt += "\n".join(lines)
```

3. **可选** — 在 `_build_recs_table()` 中加一道后置过滤，万一 LLM 不听话也直接去重：遍历 `llm_rows`，跳过名称匹配 `REJECTED_PATH` 中的条目。

### 效果

- LLM 看到 "已评估并拒绝" 章节 → 自然避开推荐
- 后置过滤兜底
- 新增拒绝条目只需追加 JSON，零代码修改
- 已缓存的结果不受影响（下次全量审计时生效）
