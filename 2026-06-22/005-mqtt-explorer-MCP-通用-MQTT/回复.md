不是 skill，是 MQTT 调试工具，能让你订阅/发布原始 MQTT 消息、看 payload 结构。

对你来说场景很窄：P1S 用的就是 MQTT 协议，但 p1s-mcp 已经把 P1S 的 MQTT 操作封装成高级工具了（状态/启停/温度/文件管理）。平时根本不需要看原始消息。

只有当 p1s-mcp 出问题、需要排查 MQTT 层时才有用——比如连不上打印机、收不到状态推送。这种场景几个月碰不到一次。

**投入产出比很低**，需要用的时候临时装一个 MQTT 客户端（比如用 `mosquitto_sub` 命令行或者装个 MQTTX GUI）就够了，不需要专门注册成 MCP server 常驻。
