## 合并方案：统一任务待办系统

### 核心设计

**一个 JSON 数据文件 + 两套入口**：

```
phone-tasks/
├── index.json       ← 统一待办数据库（当前格式扩展）
├── cli.py           ← 终端 CLI（现有扩展）
└── tasks.schema.md  ← 数据格式说明（可选）
```

### 数据格式（扩展现有 `index.json`）

```json
{
  "2026-06-22/001-重启-ops-daemon": {
    "type": "manual",
    "status": "pending",
    "summary": "重启 ops-daemon",
    "source": "手机/飞书",
    "created_at": "2026-06-22T02:02:39Z"
  },
  "2026-06-22/002-推荐-mqtt-explorer": {
    "type": "tool-recommend",
    "status": "pending",
    "summary": "MQTT 调试工具，辅助 P1S 排障",
    "source": "daily-report tool_audit",
    "tool_name": "mqtt-explorer",
    "tool_type": "MCP",
    "actions": ["accept", "reject", "defer"],
    "created_at": "2026-06-22T08:00:00Z"
  },
  "2026-06-22/003-记忆-my-memory-name": {
    "type": "memory-review",
    "status": "pending",
    "summary": "记忆条目需审核：my-memory-name",
    "source": "memory-mcp L3",
    "memory_name": "my-memory-name",
    "actions": ["keep", "edit", "delete"],
    "created_at": "2026-06-22T08:00:00Z"
  }
}
```

### `type` 三级分类

| type | 来源 | 可用操作 | 完成后动作 |
|------|------|----------|-----------|
| `manual` | 手机/飞书发的调研/操作任务 | mark-done | 状态变 completed |
| `tool-recommend` | daily_report tool_audit | accept/reject/defer | accept→安装 MCP/skill；reject→加 `tool_audit_rejected.json`；defer→标记 pending |
| `memory-review` | memory-mcp L3 深审 | keep/edit/delete | keep→保留；edit→改内容再存；delete→删记忆 |

### 两套入口

**1. 手机端（Claudetalk）** — 已有 `/phone_tasks` 改名为 `/tasks`，增强支持按 type 筛选

```
/tasks                  → 所有待办
/tasks tool             → 工具推荐待审核
/tasks memory           → 记忆待审核
/tasks manual           → 手动待办
/tasks mark-done <id>   → 标记完成
```

**2. 终端端（Claude Code skill）** — 新建 `tool-review` / `memory-review` skill

- `/tool-review`：读 `tool_audit_cache.json` 中未被处理(不在 rejected 且不在 index.json)的项目 → 逐条展示 → 选 accept/reject/defer
- `/memory-review`：调 memory-mcp 执行 L3 深审 → 逐条展示 → 选 keep/edit/delete

处理结果自动写入 `index.json`（defer 留在 pending），同时执行副作用（reject 写 `tool_audit_rejected.json`、accept 装 MCP/skill 等）。
