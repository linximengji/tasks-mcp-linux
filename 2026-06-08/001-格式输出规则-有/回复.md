从数据采集到最终邮件，整个管线的格式规则如下：

---

### 1. 入口数据（RSS → Entry）

```
interface Entry {
  title: string;       // 原标题
  link: string;        // 原文链接
  summary: string;     // 去 HTML 标签 + 去 "The post…" 尾巴，截 400 字
  source: string;      // 来源名（如 "EE Times Asia"）
  published: string;   // 日期前 16 字符
  title_cn?: string;   // 翻译/润色后的中文标题
  summary_cn?: string; // 翻译/润色后的中文摘要，截 250 字
}
```

### 2. 翻译/润色格式（DeepSeek）

**翻译 prompt 要求：**
```
[N] 中文标题 — 中文摘要(≤150字)
```
- 保留 `[N]` 序号前缀，与输入条目序号对应
- 保留专有名词（TSMC/ASML/SiC/HBM/CoWoS 等不翻译）
- 技术术语准确

**润色 prompt 额外要求：**
- 删除宣传性语言（"令人震惊""重磅""彻底改变"）
- 减少破折号使用
- 减少连接性短语（"与此同时""值得注意的是""毫无疑问"）
- 语气像行业分析师简报

### 3. 分类规则

7 个类别，优先级顺序：
1. **设备材料** — SiC/GaN 衬底、光刻胶、光刻机/刻蚀/量测设备、ASML
2. **制造工艺** — 制程节点(3nm/2nm)、FinFET/GAA、良率、国产化替代
3. **先进封装** — CoWoS/HBM/Chiplet/TSV/Hybrid Bonding
4. **芯片产品** — RISC-V、AI 加速器、硅光、CPU/GPU 微架构
5. **EDA/IC设计** — Synopsys/Cadence、UVM 验证、物理设计、RTL
6. **市场供应链** — 财报/并购/融资、市占率/扩产、制裁/出口管制
7. **产业政策** — CHIPS Act、大基金、补贴、十五五

**匹配方式**：`text.includes(kw)` 不区分大小写，**first-match-wins**，匹配到就归类，继续下一条。7 个类别都没命中 → "其他"。

### 4. 高亮/重点条目

掩模版相关（mask/photomask/reticle）命中任意关键词 → 从所属分类提出，集中到独立的 **"🔥 掩模版重点"** 区，用琥珀色 (#f59e0b) 视觉区分。

### 5. HTML 报告排版

| 区块 | 内容 |
|------|------|
| Header | 标题 + 日期 + 统计（共采集 N 条，每个分类 X 条） |
| 🔥 掩模版重点 | HIGHLIGHT_KEYWORDS 命中的条目，金色边框 |
| 7 个分类（CAT_ORDER 顺序） | 各分类条目，带颜色主题/图标 |
| 其他 | 最多展开 8 条，超过的显示 "+ N 条其他资讯" |
| 💡 情报摘要 | 每类取第一条标题（截 35 字），底部显示数据源 |
| Footer | 生成时间戳 + Claude Code 署名 |

每个条目渲染：
```
h3: [链接] title_cn || title
p:  summary_cn || summary (截 280 字)
footer: 来源 | 日期 | 查看原文 →
```

"其他"分类条目截断上限 8 条，其余收起。其余分类不限量。

### 6. 输出产物

- **reports/report-{YYYY-MM-DD}.html** — 完整 HTML 邮件
- **reports/data-{YYYY-MM-DD}.json** — `{ classified: Record<category, Entry[]>, total: number }`，用于隔天去重
- 通过 163 SMTP 发送 HTML 邮件

### 7. 去重规则

- **跨源去重**：首 50 字符 title 相同只留一条
- **跨天去重**：回溯 3 天的 data-*.json，已出现过的 title（首 50 字符）不再收录
