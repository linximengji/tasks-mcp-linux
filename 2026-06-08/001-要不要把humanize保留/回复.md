可以在 prompt 里解决。在 system prompt 加一句风格约束，效果等同于内置 humanize，不用多跑一次 API。

**对比：**

| | 方案 A: 保留 humanize (两步) | 方案 B: prompt 内置 (一步) |
|---|---|---|
| API 调用 | 翻译 → 润色，两次 | 一次 |
| 延迟 | 翻倍 | 不变 |
| 成本 | 翻倍 | 不变 |
| 效果 | 专门优化过的润色 prompt | 同等约束写到合并 prompt 里 |

**具体做法：** 在 llm_processor 的 system prompt 里加：

> 中文标题和摘要要求：信息密度高、简洁、符合中文新闻语感。禁止翻译腔（如"被进行""有着""在……方面"等被动/冗余句式）。读起来像财新/晚点的短讯，不像机翻。

这样 DeepSeek 一步完成翻译+润色，不需要第二步。如果实际跑下来发现 AI 味还是重，再单独调 prompt 就行，不需要改管线结构。
