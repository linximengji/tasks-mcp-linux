目前飞书渠道的收发能力不对称：

**✅ 接收** — 支持五种类型：
- `text` 文字
- `image` 图片（下载到本地，LLM 多模态用 base64 传入）
- `post` 富文本（含文字+嵌套图片）
- `file` 文件（下载到本地，50MB 上限）
- `audio` 语音（走飞书 ASR API 转文字）

**❌ 发送** — 仅支持纯文本（`msg_type: 'text'`）
- 不能发送图片、文件、富文本卡片
- `sendMessage()` / `sendMessageWithId()` / `editMessage()` 全部走的 `sendTextMessage()`

如果你想加发送文件/图片的能力，需要：
1. 在 `sendMessage()` 里根据内容类型选择 `msg_type: 'file'` / `'image'` / `'post'`
2. 调用飞书上传文件 API（`/im/v1/files`）拿到 `file_key` 再发
3. 扩 `Channel` interface 的 `sendMessage` 签名（目前只传 `string content`）

要加这个功能？需要的话先 brainstorm 方案。
