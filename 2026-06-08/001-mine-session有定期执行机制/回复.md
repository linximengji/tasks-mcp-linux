## 1. 定期执行

`daily_pipeline.bat` 已经有了，问题是"谁来触发它"。

**方案 A：Windows Task Scheduler（推荐）**
——原生、可靠、重启后还在、无需额外依赖

```powershell
# 创建每天 23:00 执行的任务
$action = New-ScheduledTaskAction -Execute "cmd.exe" `
  -Argument "/c D:\ClaudeProjects\claudetalk\daily_pipeline.bat"
$trigger = New-ScheduledTaskTrigger -Daily -At 23:00
$principal = New-ScheduledTaskPrincipal -UserId "34971" -RunLevel Limited
Register-ScheduledTask -TaskName "claude-session-mining" `
  -Action $action -Trigger $trigger -Principal $principal
```

**方案 C：追加到已有日报管线**

## 2. 输出建议

当前输出是每轮 4 个 JSON 文件（tool_stats / patterns / knowledge / memory），每次跑都写全新文件。**JSON 本身不是「产出」**，要让数据有用需要一条消费链路。

几个方向（从轻到重）：

**① 每日摘要文本** — 轻量、可读
每次 mine 后自动生成一段 Markdown，包含：
- 今天处理了多少新 session，entrypoint 分布
- top 3 的 tool chain 变化
- 显著的 knowledge 片段（heuristic tags 命中最多的类型）
- 异常（高 error 率 session）

可以直接推到飞书消息。daily_pipeline.bat 末尾加一步即可。

**② 滚动聚合视图** — 基于 SQLite
当前 mine 是一次性的 JSON 快照。如果要「本周 vs 上周」「tool 使用趋势」，需要把 mining 结果存回 SQLite（新建 `mine_snapshots` 表），然后用 SQL 做时间序列聚合。

**③ 飞书推送** — 把日报摘要推到飞书群
你已经有 feishu MCP 和 notify 工具可用。每天 mine 完发一条 markdown 消息到群里。

**④ 记忆入库** — 把 high-value 片段写入 memory-mcp
每天跑完把 LLM 提炼过的 knowledge 和 memory 条目写入 MCP memory_store，之后任何 session 都能通过 `memory_recall` 查到。
