.bat 弹 cmd 窗口，换成 `.ps1`，Task Scheduler 用 `-WindowStyle Hidden`：

& $python D:\ClaudeProjects\claudetalk\collect_sessions.py

```powershell
schtasks /create /tn "claude-mining" /tr "powershell.exe -NoProfile -WindowStyle Hidden -File D:\ClaudeProjects\claudetalk\daily_pipeline.ps1" /sc daily /st 23:00 /f
```

完全不弹窗。日志追加写到文件而不是 console。

## 2. 方案 4 消费：什么值得存，怎么存

memory-mcp 底层是 `D:/ClaudeProjects/.claude/memory/*.md` 文件，mined output 里不是所有东西都值得进去。**筛选标准**：将来 CC session 里 `memory_recall` 能派上用场的信息，才是好记忆。

### 值得存 / 不值得存

| mined 数据 | 是否值得存 | 原因 |
|---|---|---|
| tool_stats | ❌ | 工具频度变化太快，没 recall 价值 |
| patterns（tool chains / 时长分布） | ❌ | 抽象统计，对后续 session 无帮助 |
| knowledge（heuristic tags） | ❌ | 太多太杂，recall 全是噪声 |
| knowledge（LLM-refined） | ✅ 精选 | 单条有信息量（"你解决了 X 问题，方案是 Y"），domain 可归项目 |
| memory（session summaries） | ⚠️ 少量 | LLM 摘要可以作为项目进展记录，但每条只存最有意义的 |

### 存放方式

语法就是 `name + description + content + domain` 的 .md，直接写入 `D:/ClaudeProjects/.claude/memory/` 即可，不调 MCP 协议。server.py 的 validation 逻辑可以独立 import 或用简单的文件名去重。

### 只存 LLM-refined

当前每日跑带 `--skip-llm` 的日常 mine 是没有 LLM 精炼的。如果走方案 4，每日跑需要去掉 `--skip-llm`，或者加一个新 phase 专门处理。LLM 精炼的成本约 10-20 秒 × flash 模型，几乎免费。

### 消费链路

```
采集 → mine（含 LLM） → 过滤优质条目 → 写入 memory/
                                           ↓
                                   下次 CC session 中 memory_recall 命中
```

你的场景是 solo 开发者，项目数 5-6 个（claudetalk、semi-report、blender-mcp、p1s-mcp、model-proxy），每条知识归到对应 domain。大致的实际效果：

- 每天 mine 完产生 1-3 条 memory，如 `"blender-mcp: 解决了 TCP 断线重连的 5 次重试策略"`
- 哪天再搞 blender-mcp 时，`memory_recall("blender-mcp tcp")` 自动命中
- 如果天天都存同样的东西（去重不通过），就不存
