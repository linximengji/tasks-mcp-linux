对。Markdown 文件适合人类编辑的知识条目，但 session mining 的产出本质是**结构化分析数据**——SQLite 更合适。

关键区别：

| | Markdown Memory | SQLite |
|---|---|---|
| 用途 | 人类编辑的确定性知识 | 挖掘出的模式、统计、趋势 |
| 查询 | grep / 文件名 | SQL（跨时间、跨项目聚合） |
| 跨 session 分析 | 很难 | 天然支持 |
| 可编辑 | 人工修正 | 只读分析结果，源数据可追溯 |

**建议两层的架构：**

1. **SQLite** — `memory/mined_sessions.db`，存 mining 原始产出
   - 表：`sessions`（session_id, project, timestamp, duration, commands...）
   - 表：`patterns`（pattern_type, description, confidence, source_sessions...）
   - 表：`errors`（error_type, file, frequency, first_seen, last_seen...）
   - 可以做：过去 30 天最高频的失败模式是什么？哪个项目的 session 在变长/变短？

2. **Markdown Memory** — 只存 SQL 分析后确认的"应该记住的结论"
   - 比如从 SQL 发现 "claudetalk 项目 80% 的失败来自 MQTT 重连超时" → 写入 `projects/claudetalk/mqtt-timeout-pattern.md`

这样 SQLite 做纵向分析，Markdown 存精选结论。两者不互斥。
