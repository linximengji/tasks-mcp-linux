### Assumption Inventory Prompt

发给 AI 的 preamble，编码前执行：

```
在开始编码之前，先完成 Assumption Inventory：

## 1. 目标复述
用 1-2 句话复述你的理解，确认我们目标一致。

## 2. 假设清单（至少 6 条）
按以下维度列出你隐含的假设，每条标注风险等级（Low/Medium/High）：

Product / UX
- 用户是谁？他们期望什么交互方式？
- 什么算"完成"？验收标准是什么？

Data / Edge Cases
- 数据从哪里来？什么格式？大体量？
- 空状态、错误状态、极端输入时行为？

Security / Permissions
- 谁有权限？认证从哪里来？
- 敏感数据是否需要脱敏？

Performance / Integration
- 预期的延迟/吞吐量？
- 依赖哪些外部服务？不可用时怎么降级？

## 3. 澄清问题（最多 5 个）
列出你需要确认的关键问题，按重要性排序。
如果没有答案，声明 safe default。

## 4. Safe Defaults 声明
对每个未确认的问题，说明你选的默认行为及理由。
```

---

### 与 SDD 的集成方式

在 **Brainstorming → EnterPlanMode** 之间插入此步骤，`.plan.md` 的每个 task 标注对应的假设编号（如 `[假设 2.1] 分页默认 20 条`），review 时检查假设是否漂移。整体开销 < 3 分钟。
