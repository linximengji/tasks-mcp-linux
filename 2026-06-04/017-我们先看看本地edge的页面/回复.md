几种技术路径，适用场景不同：

## 1. Cookie 提取 + API 调用

最轻量。从 Edge profile 目录提取 DeepSeek 的 session cookie，然后用 HTTP 请求直接调 DeepSeek 的接口拿对话数据。

**问题**：Edge 的 cookies 在 SQLite 里且加密（Windows DPAPI），解密需要额外代码。而且 DeepSeek 的 cookie 有效期通常很短。

## 2. Playwright + 持久化 Edge profile

用 Playwright 启动浏览器，复用 Edge 的用户目录 → DeepSeek 登录态自动继承 → 直接调页面 API 或爬 DOM。

```python
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch_persistent_context(
        user_data_dir=r"C:\Users\34971\AppData\Local\Microsoft\Edge\User Data",
        channel="msedge",
        headless=True,
    )
    page = browser.new_page()
    page.goto("https://chat.deepseek.com")
    # 已登录，直接可用
```

**问题**：Edge 在运行时会锁定 user_data_dir，不能同时复用。需要关掉 Edge 或复制 profile。

## 3. Edge Remote Debugging

启动 Edge 时加 `--remote-debugging-port=9222`，然后用 CDP 协议远程控制：

```powershell
# 启动 Edge 并开启调试端口
& "C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" --remote-debugging-port=9222

# 然后 Playwright/Puppeteer 连接上去
browser = p.chromium.connect_over_cdp("http://localhost:9222")
```

**优点**：Edge 正常运行，不锁 profile。**这是最实用的方案**。

---

**关键问题**：DeepSeek 的网页端数据实际是服务端 API 返回的。不管哪种方式，只要拿到浏览器里有效的 session token，就可以直接 POST 到 DeepSeek 的 API 拉对话列表。

**我建议的路径**：方案 3（Remote Debugging），你不需要关闭当前 Edge，直接用 Playwright 连上，访问 `chat.deepseek.com`，拦截 XHR 拿到 session token，然后就可以用 HTTP 批量拉数据了。
