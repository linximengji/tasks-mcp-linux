# Claude Code Workflow 使用指南

Workflow 是多 Agent 编排工具，让你用脚本控制多个子 Agent 并行/流水线执行。核心是 `Workflow` 工具调用，传入一个 JS 脚本（不是 TypeScript）。

## 基本结构

```js
// 每个脚本必须以 meta 开头（纯字面量，不能用变量/函数）
export const meta = {
  name: 'review-changes',
  description: '审查代码变更',
  phases: [
    { title: '审查', detail: '多维度代码审查' },
    { title: '验证', detail: '对抗性验证发现' }
  ]
}

// 脚本体可用的函数：
// agent()     — 启动一个子 Agent
// parallel()  — 并行执行多个任务（屏障模式，等全部完成）
// pipeline()  — 流水线模式（不等待同阶段全部完成）
// phase()     — 标记阶段
// log()       — 输出进度信息
// args        — 外部传入的参数
```

## 核心概念

### 1. agent() — 启动子 Agent

```js
// 纯文本结果
const result = await agent('分析这个文件中的 bug', { label: 'bug-finder' })

// 结构化结果（用 JSON Schema）
const bugs = await agent('找出所有 SQL 注入漏洞', {
  schema: {
    type: 'object',
    properties: {
      findings: { type: 'array', items: { type: 'object' } }
    }
  }
})
// bugs 是已校验的对象，无需手动 parse
```

### 2. parallel() — 并行执行（有屏障）

```js
// 全部任务完成后才返回
const results = await parallel([
  () => agent('审查安全性', { schema: REVIEW_SCHEMA }),
  () => agent('审查性能',   { schema: REVIEW_SCHEMA }),
  () => agent('审查可维护性', { schema: REVIEW_SCHEMA })
])
// results = [安全结果, 性能结果, 可维护结果]
```

用 `parallel` 的场景：**后一步需要前一步全部结果**才能进行（如去重、合并、统计总数）。

### 3. pipeline() — 流水线执行（无屏障）

```js
// 每个 item 独立走完所有阶段，不等待同阶段别的 item
const results = await pipeline(
  ['src/auth.py', 'src/api.py', 'src/db.py'],  // items
  // Stage 1: 分析
  (file) => agent(`分析 ${file}`, { schema: ANALYSIS_SCHEMA }),
  // Stage 2: 修复（上一个阶段的返回值 → 这个阶段）
  (analysis, file) => agent(`修复 ${file}: ${analysis.issues}`, { schema: FIX_SCHEMA })
)
```

`pipeline` 比 `parallel` 更快 —— item A 在 stage 2 时，item B 可以还在 stage 1。**默认选 pipeline**。

### 4. phase() — 标记阶段

```js
phase('审查')
const reviews = await parallel([...])  // 这些 agent 归入"审查"组

phase('验证')
const verified = await parallel([...]) // 这些 agent 归入"验证"组
```

## 常用模式

### 对抗性验证（反证模式）

```js
// 一个 Agent 提发现，多个 Agent 独立尝试反驳
const claim = { title: '潜在 OOM', file: 'worker.py', line: 42 }

const votes = await parallel(
  Array.from({ length: 3 }, () => () =>
    agent(`尝试反驳: ${claim.title}。不确定就判 refuted=true。`, {
      schema: { type: 'object', properties: { refuted: { type: 'boolean' } } }
    })
  )
)
const survives = votes.filter(Boolean).filter(v => !v.refuted).length >= 2
```

### 多维度审查

```js
const DIMENSIONS = [
  { key: 'bugs',     prompt: '找出 bug，尤其是并发和边界条件' },
  { key: 'security', prompt: '安全审计：注入、越权、敏感信息泄露' },
  { key: 'perf',     prompt: '性能问题：N+1、不必要分配、阻塞 IO' },
]

// pipeline: 每个维度审完立刻开始验证，不等别的维度
const results = await pipeline(
  DIMENSIONS,
  (d) => agent(d.prompt, { label: `review:${d.key}`, phase: '审查', schema: FINDINGS }),
  (review) => parallel(
    review.findings.map(f => () =>
      agent(`验证 ${f.title}`, { phase: '验证', schema: VERDICT })
    )
  )
)
```

### 循环直到收敛

```js
const bugs = []
let dry = 0
while (dry < 2) {
  const result = await agent('继续找没发现的 bug', { schema: BUGS })
  if (!result.bugs.length) { dry++; continue }
  dry = 0
  bugs.push(...result.bugs)
}
```

## 什么时候用

| 场景 | 方式 |
|------|------|
| 改两三个文件 | 直接动手，不用 workflow |
| 审查整个项目的安全性 | workflow：多维度 + 对抗验证 |
| 同时分析 10+ 文件 | workflow：pipeline 流水线 |
| 大规模迁移/重构 | workflow：发现 → 迁移 → 验证 多阶段 |

## 限制

- 最多 16 个并发 Agent
- 整个 workflow 生命周期最多 1000 个 Agent 调用
- 脚本是纯 JS，不能用 `Date.now()` / `Math.random()` / `new Date()`
- 子 Agent 不能递归调 workflow
