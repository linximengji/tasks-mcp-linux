## 问题 1：模型切换时数据包格式对齐

先看当前的路由矩阵。三个 provider 的实际转发路径：

| 提供方 | 当前模型 | 协议 | CC 发出格式 | 格式处理 | 风险 |
|--------|---------|------|------------|---------|------|
| **DeepSeek** | v4-pro, v4-flash | Anthropic | Anthropic | `sanitize_for_deepseek` 删无关字段，直发 | 🟢 |
| **Aliyun MAAS** | qwen3.7-max, qwen3.6-plus | Anthropic | Anthropic | **不处理，body.copy() 直通** | 🟢 |
| **DashScope** | qwen3.7-max-ds, qwen3.6-plus-ds (quota_backup) | OpenAI | Anthropic | `anthropic_to_openai()` 转换 | 🟡 |
| **火山引擎** | doubao-1.5-vision-pro | OpenAI | Anthropic | `anthropic_to_openai()` 转换 | 🟡 |

### 切换路径格式变化

```
DeepSeek ↔ qwen(MAAS):    无变化，两边都是 Anthropic 直通    → 🟢
qwen(MAAS) → qwen-ds:     Anthropic → OpenAI 转换            → 🟡 quota_backup 路径
DeepSeek → doubao-vision:  Anthropic → OpenAI 转换 + 视觉    → 🟡 跨协议
```

### 三个已知盲区

**盲区 1：thinking 块 → qwen MAAS**

CC 可能带 `thinking`/`redacted_thinking` 块（来自之前 assistant 回复）。路由到 qwen MAAS 时：

```
body.copy() → 直通发送 → qwen MAAS 端如果不知道 thinking 块类型 → 422
```

当前代码路径：L2 → `deepseek? sanitize` / `anthropic? 不处理`。`sanitize_for_deepseek` 有白名单包含 `thinking`/`redacted_thinking`，所以两个路径都没过滤。**但这个在实际运行中没触发过**，因为 CC 对话历史里的 thinking 块只在确认模型支持 thinking 时才出现。

**盲区 2：streaming 下 OpenAI SSE → Anthropic SSE 转换的状态机**

`handlers.py` 第 130-215 行，转换维护了三个状态变量：
- `ti` — text content_block index
- `tbs` — tool_call index 映射表
- `nbi` — 下一个 content_block index

问题：`content_block_start` 在首次遇到 tool_call index 时就发了，此时 id 和 name 可能为空（OpenAI 分 chunk 发）。虽然有后续 delta 补充，但 Claude Code 客户端对"延迟填充的 content_block_start"的容忍度取决于客户端实现。

**盲区 3：DeepSeek sanitize 的时序——这个是正确的**

```
路由 → 确定最终模型 → sanitize(如果是 deepseek) → 转发
```

sanitize 只在确定路由到 deepseek 后才执行。不会出现"sanitize 完了才切模型"的情况。
