你说得对。当前所谓的"Allocator"其实是个空壳：

```
ALLOCATOR.compute_ratio(rem, total, days)   → 0.0~1.0    (telemetry 级别的计算)
ALLOCATOR.should_route_to_max(cplx, id, r)  → True/False (纯 hash)
_budget_adjust(cplx, id, r)                  → model_name (调 should_route_to_max)
```

三个函数唯一的关联是 ratio 作为参数传递。[Allocator 本身](D:\ClaudeProjects\proxy\proxy_lib\allocator.py:5-51) 只是两个独立方法的容器，`compute_ratio` 本质是预算计算（放 telemetry 没区别），`should_route_to_max` 是个 hash 门，它们之间没有数据共享，没有状态，没有分配逻辑。改成一体的 `MultiModelAllocator` 刚刚好：

```
MultiModelAllocator
  ├── _mapping_table      # (complexity, task_type) → target_model + min_ratio + fallback
  ├── compute_ratio()     # 预算计算（原有，保持不变）
  ├── select()            # 核心：接收 complexity+task_type+req_id → 走矩阵查表 → hash门 → 返回 model_name
  └── _hash_gate()        # 概率门的通用实现
```

`_budget_adjust()` 这个薄包装函数直接取消，调用方改为：

```python
# 之前
adjusted = _budget_adjust(complexity, get_req_id(), ratio)

# 之后
result = ALLOCATOR.select(complexity, task_type, get_req_id(), ratio)
if result:
    model_name = result  # 非 None → 走 TP 模型
```
