## 调查结果与初步方案

### 当前利用率分析

**83,766 条请求的模型分布：**

| 模型 | 请求数 | 占比 | 状态 |
|------|--------|------|------|
| deepseek-v4-flash | 56,877 | 67.9% | 主力 |
| deepseek-v4-pro | 25,178 | 30.1% | 主力 |
| qwen3.7-max | 1,135 | 1.4% | **TokenPlan 最大消耗** |
| qwen3.6-plus | 120 | 0.14% | 几乎没用 |
| doubao-1.5-vision-pro | 96 | 0.11% | 仅图片时触发 |
| qwen3-coder-plus | 3 | 0.004% | **基本废弃** |
| 其余（GLM/Kimi/MiniMax/…） | 0 | 0% | **完全没接** |

**核心问题：**
1. TokenPlan 买了 25,000 credits 套餐（含 Qwen3.7-Max、Kimi-K2.6、GLM-5.1、MiniMax-M2.5、DeepSeek-V4-Pro 等），但只用了 Qwen3.7-Max（1.4% 流量），其他模型零利用
2. 路由架构只有 4 个 tier 槽位，没有 task-type-aware（代码/推理/创意/数学）的分类
3. qwen3.7-plus 和 qwen3-coder-plus 已注册但从不路由过去
4. Allocator 当前的"moderate→qwen3.7-max 概率分流"无法覆盖多模型场景

### 建议分步方案

**Phase 1 — 扩展 Tier 体系（低风险，立即价值）**

从 4 tier 扩展到 7 tier，覆盖 TokenPlan 已有模型：

```
flash   → deepseek-v4-flash (不变)
pro     → deepseek-v4-pro   (不变)  
max     → qwen3.7-max       (不变，最贵 5x，仅复杂任务)
vision  → doubao-1.5-vision-pro (不变)
coder   → qwen3-coder-plus  (新增，TokenPlan 3x)
plus    → qwen3.7-plus      (新增，已注册未用，中等成本)
cheap   → qwen3.6-flash     (新增，DashScope 按量计费)
```

- L1 规则增加：`_has_code_indicators()` → 路由到 coder 层而非 pro
- L2 classifier 输出增加 task_type 维度：`complexity + task_type`
  - code → coder tier
  - reasoning → pro/max
  - creative → plus tier
  - factual → flash
- `@tag` 别名扩展：`@coder`, `@plus`, `@cheap`, `@kimi`, `@glm`

**Phase 2 — 精算 TokenPlan 配额**

当前 Allocator 只控制 moderate→max 一个维度。需要扩展到多模型：

```
每个 tier 可以独立配置：
- 是否走 TokenPlan credits / DashScope 按量 / 免费 API
- credits 消耗倍数（qwen3.7-max = 5x, coder-plus = 3x, plus = 1x）
- 按余额自动切换降级链
```

示例降级链设计：
```
coder 链: qwen3-coder-plus (TP) → deepseek-v4-pro (API) → qwen3.6-flash (DS)
plus 链: qwen3.7-plus (TP)    → deepseek-v4-pro (API) → qwen3.6-flash (DS)
```

**Phase 3 — Sub-agent 模型路由优化**

当前 sub-agent 固定走 flash/pro。改为：
- 代码任务 → coder tier
- 简单搜索 → cheap/flash
- 复杂分析 → pro

**Phase 4 — 引入 Kimi/GLM/MiniMax（可选）**

只有 Phase 1-2 稳定后才引入：
- Kimi-K2.6 → 超长上下文/推理 (目前被 qwen3.7-max 覆盖，价值低)
- GLM-5.1 → 中文创作 (目前被 deepseek 覆盖，价值低)
- MiniMax-M2.5 → 创意文本 (目前被 deepseek 覆盖，价值低)

我的判断：Phase 1-2 就能覆盖 80% 价值，Phase 3-4 边际收益有限。可以**先 Phase 1 快速落地**（改 router.py + litellm_config.yaml），实际运行几天看效果再决定是否继续。
