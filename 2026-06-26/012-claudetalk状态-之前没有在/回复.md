`check_claudetalk` 改过——旧版在 PID 存活后会额外做 TCP 连 `:9877`，但 claudetalk (cli.js) 本身不监听端口，所以端口检查必定超时，返回 `"zombie"`。`latest.json` 里一直是 zombie 就不会在 dashboard 上显示为正常状态。

当前的版本（`claudetalk.py:37-67`）只做两件事：

1. crash marker 检测
2. PID 文件 + tasklist 确认进程活着

**砍掉了 socket connect(9877) 这一步**，因为那个端口是 mcp-standalone.js 的，而 `check_mcp_server`（`claudetalk.py:70-92`）已经专门负责检查它了。

所以现在 claudetalk 进程活着就直接返回 `"up"`，写入 `latest.json`，dashboard 读出来自然就展示了。

昨天 `latest.json` 就能看到两条记录并存，互不干扰：

```json
"claudetalk": { "status": "up", "pid": 5056 },
"mcp_server": { "status": "up", "port": 9877 }
```
