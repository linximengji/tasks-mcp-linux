## 现状诊断

当前 `pipeline.ts` 的关键流程：

```
RSS → LLM 筛选 → HTML + JSON → 发邮件 → 写 sentinel
```

**两个薄弱点**：

1. **LLM 返回空结果不阻断**：7 个分类全空时，HTML 生成"无内容"页面，照常写文件、发邮件、写 sentinel
2. **sentinel 在发邮件后写**：只要 `sendEmail` 返回 `true`（邮件因幂等检查跳过也算 true），sentinel 就写了，管线标记"已完成"

刚加的**质量门禁**只解决一部分——保留率 < 15% 重试一次，但重试后仍为 0 仍会发空邮件。

## 方案

**核心思路**：在"生成输出"和"发送邮件"之间加两道检查：

```
RSS → LLM 筛选 → [质量门禁: 全空→重试→仍空→关键词回退] → HTML → [内容检查: 全空→不发送] → 邮件 → sentinel
```

**具体改动**（都在 `pipeline.ts`）：

| # | 改动 | 说明 |
|---|------|------|
| 1 | **质量门禁增强** | LLM 全空 → 重试一次 → 仍空 → 自动 fallback 到 `--skip-llm`（关键词分类），保证至少有一点内容 |
| 2 | **内容检查** | 发邮件前检查 `classified` 是否全空 → 全空则打日志、不写 sentinel，退出码非 0 |
| 3 | **sentinel 条件收紧** | 只有 `!dryRun && ok && retained > 0` 才写 sentinel |

**不动**的地方：
- 参数语义不变（`--skip-llm` 仍是手动回退，质量门禁是自动回退）
- LLM 重试逻辑不变
- 邮件发送不变
