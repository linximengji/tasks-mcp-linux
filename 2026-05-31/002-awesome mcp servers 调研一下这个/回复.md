## awesome-mcp-servers 调研

### 核心仓库

**punkpeye/awesome-mcp-servers** 是社区公认的权威 MCP 服务器精选列表，GitHub awesome list 格式，收录 **200+** 个 MCP 服务器，按 **30+ 类别** 组织。

每个条目标准化标注：
- 语言图标（🐍 Python / 📇 TypeScript / 🏎️ Go / 🦀 Rust / ☕ Java / #️⃣ C#）
- 部署范围（☁️ 云端 / 🏠 本地 / 📟 嵌入式）
- 平台支持（🍎 macOS / 🪟 Windows / 🐧 Linux）

### 主要分类

| 大类 | 子类 |
|------|------|
| **数据访问** | 数据库（50+ 实现：PostgreSQL、MySQL、MongoDB、Neo4j、Pinecone、BigQuery...）、文件系统、数据平台 |
| **自动化** | 浏览器自动化、命令行/Shell、代码执行、Coding Agent |
| **集成** | 云平台（AWS/GCP/Azure）、通讯/社交、DevTools、版本控制 |
| **智能** | 知识/记忆、搜索/检索、监控/可观测性、安全/威胁情报 |
| **垂直领域** | 金融、生物医药、法律、游戏、艺术/媒体 |

另有 **框架/工具**（构建 MCP Server 的库）、**客户端**（Claude Desktop、IDE 插件等）、**教程资源** 等配套内容。

### 2026 MCP 注册中心生态

除了 awesome list，2026 年形成了四个主要注册中心：

| 注册中心 | 规模 | 提交方式 | 特点 |
|----------|------|----------|------|
| **[mcp.so](https://mcp.so)** | **20,222** 个 | 自助表单 | 最大，Claude Desktop/Cursor 用户首选，直接渲染 config 代码片段 |
| **[smithery.ai](https://smithery.ai)** | - | CLI 发布 | 含 OAuth 托管 + agent.pw 认证层，框架向 |
| **[glama.ai/mcp](https://glama.ai/mcp)** | - | 人工审核 | 偏企业级，对 LLM 训练语料库有影响力 |
| **punkpeye/awesome-mcp-servers** | 200+ | PR | 唯一有 DoFollow 外链的，LLM 和开发者都会爬取 |

### 对我们有意义的方向

与我们现有项目相关的类别：

1. **3D/Blender** — 已有 blender-mcp，可以看看 awesome list 里有没有更丰富的 3D 工具链
2. **IoT/打印机** — p1s-mcp 是我们自建的，社区可能冒出其他 3D 打印机/硬件 MCP Server
3. **飞书生态** — 31 个 Lark skill 如果要做成 MCP Server 对外发布，可以走 mcp.so + smithery 两条渠道
4. **半导体数据管线** — semi-report 的数据源可以探索是否有数据库/API 类 MCP Server 能替代手动 RSS

### 结论

awesome-mcp-servers 本身是个 **元数据目录**，不是代码仓库——它的价值在于发现和标准化。2026 年 MCP 生态已经成熟到有专门的注册中心竞争，mcp.so 以 2 万个 server 的规模遥遥领先。如果我们要发布自己的 MCP Server（如 blender-mcp、p1s-mcp），同时提交到 mcp.so + smithery + awesome list 三个渠道就能覆盖绝大多数发现场景。
