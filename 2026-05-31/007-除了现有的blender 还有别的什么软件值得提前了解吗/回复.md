## 值得关注的 CLI-Anything 软件（按对我们的价值排序）

### Tier 1 — 直接增益现有工作流

#### FreeCAD / OpenSCAD（参数化 3D CAD）

**为什么跟我们有关**：你用 P1S 做 3D 打印。Blender 是 mesh 建模（适合有机形状、雕刻），FreeCAD 是参数化 CAD（适合机械零件、尺寸约束、装配体）。

Blender → 3D 打印的典型痛点：做个带精确螺纹孔的支架，Blender 里用 Boolean modifier 很痛苦，FreeCAD 一个 sketch + pad + hole 操作就完事。

CLI-Anything 把 FreeCAD 的 sketcher/part design/assembly 全变成结构化命令，Agent可以直接生成参数化模型，导出 STL，然后通过 p1s-mcp 打印。

OpenSCAD 更极端——纯代码驱动，天然适配 AI Agent，CLI 覆盖度会更高。

```
自然语言 → CLI-Anything FreeCAD → STL → p1s-mcp 打印
          ↑ 新能力                 ↑ 现有
```

#### Inkscape（矢量图形）

**为什么**：生成 SVG → 导入 Blender 做 extrusion → 3D 打印。或者做 laser cutting 模板、logo 设计。

CLI-Anything 的 Inkscape 命令操作 SVG XML 直接层，绕过 GUI。对 Agent 来说比"操作 Inkscape GUI"可靠得多，因为 SVG 是纯文本格式。

与 image-gen 配合：image-gen 生成位图 → Inkscape 矢量化 → Blender 建模 → 3D 打印。一条龙。

#### Draw.io（架构图/流程图）

**为什么**：你经常需要画架构图（proxy 路由流程、pipeline 结构、项目计划）。当前只能靠文字描述或手画。

CLI-Anything 的 Draw.io 操作 mxGraph XML，Agent 可以直接生成标准化的图表。以后讨论架构时产物自动变成可视化，不用另外画。

### Tier 2 — 扩展能力边界

#### ComfyUI（本地 AI 图像生成工作流）

**为什么**：你现在 image-gen 走 DashScope API（云端 qwen-image-2.0-pro）。ComfyUI 跑本地模型（Stable Diffusion/FLUX），没有 API 费，且工作流可定制（ControlNet、IP-Adapter、inpaint 节点链）。

CLI-Anything 可以把 ComfyUI 的节点图变成可编程的——Agent 自动连接节点、调参数、跑 batch。比在 ComfyUI 里手动拖节点快得多。前提是你有一张能跑 SD 的显卡。

#### OBS Studio（录制/推流）

**为什么**：场景控制 + 录制。如果你需要录制 demo 视频、screenshot 工作流程、或者定时录制某些画面。

CLI-Anything 操作 OBS 的 JSON scene collection + obs-websocket 后端。Agent 可以切场景、开始/停止录制、截图，全程自动化。

#### LibreOffice（文档生成）

**为什么**：semi-report 目前生成 HTML 报告。如果需要输出 PDF/DOCX/PPTX，LibreOffice 的 headless 模式 + CLI-Anything 可以自动排版、插入图表、导出。

比 Python reportlab 手动排版灵活得多——Agent 生成 ODF XML，LibreOffice 渲染。

### 但这张表不只是功能清单

CLI-Anything 的价值不是"多了几个软件的可编程接口"，而是**把 Agent 的触角从纯代码领域伸到了桌面软件领域**。举几个具体组合：

```
现有链路                          CLI-Anything 扩展后
──────────────────────────────────────────────────
blender-mcp 建模                  FreeCAD 参数化建模 → blender 渲染
p1s-mcp 打印                     Inkscape 设计 → blender 挤出 → p1s 打印
image-gen 出图                  ComfyUI 本地工作流 → 复杂节点 → 批量出图
semi-report HTML 报告           LibreOffice → 排版 → 导出 PDF/PPTX
文字描述架构                    Draw.io → 可视化架构图
```

最值得先试的是 **FreeCAD+Draw.io**。FreeCAD 打通了"参数化建模 → 3D 打印"这条路，Draw.io 解决了"讨论架构时没有图"的长期痛点。这两个 ROI 最高，而且是我们现有能力覆盖不到的盲区。
