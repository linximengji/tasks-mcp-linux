### 原始 transcript（CC 自动写入的 JSONL）

每行一条记录，8 种类型：

| 类型 | 内容 |
|------|------|
| `user` | 用户消息全文、cwd、git branch、entrypoint、timestamp、version、sessionId |
| `assistant` | thinking 块全文、text 回复全文、tool_use(name + 参数)、model 名 |
| `attachment` | deferred tools delta（工具定义注入） |
| `error` | 错误类型、错误消息 |
| `system` | compaction 元数据、警告、info 日志 |
| `mode` | 权限模式切换 |
| `last-prompt` | 发给模型的最后一条 prompt 全文 |
| `ai-title` | AI 生成的会话标题 |
| `queue-operation` | context compaction 注入的摘要 |

### SQLite index (`collect_sessions.py`) 从 JSONL 提取

**sessions 表：**
```
id, cwd, git_branch, entrypoint, start_time, end_time,
user_msgs, assistant_msgs, duration_sec, has_errors,
file_path, file_size, file_mtime, processed_at
```

**turns 表：**
```
session_id, turn_idx,
user_text (前 2000 字符),
asst_text (前 3000 字符),
thinking_text (前 2000 字符),
has_thinking,
tool_names (逗号分隔),
tool_calls,
error_count,
error_msgs (前 500 字符),
timestamp
```

**tool_stats 表：**
```
session_id, tool_name, call_count
```

### mine JSON (`mine_sessions.py`) 从 SQLite 精选

`_build_session_entry()` 从 turns 聚合后输出：

```python
{
    "session_id":     sid[:8],        # 短 ID
    "full_id":        sid,            # UUID 全量
    "date":           start_dt,       # "YYYY-MM-DD HH:MM"
    "entrypoint":     ep,             # cli/sdk-cli
    "task_hint":      first_ask[:200],# 第一条用户消息
    "turns":          len(turns),
    "tool_calls":     total_calls,
    "unique_tools":   len(tools_set),
    "tools":          sorted(tools_set),
    "thinking_pct":   round(...),
    "has_thinking_content": bool,
    "duration_s":     int,
    "has_errors":     bool,
    "error_count":    int,
    # LLM 追加字段:
    "llm_summary":    "一行中文摘要",  # deepseek-v4-flash 生成
}
```

LLM 的 `_llm_summarize_batch()` 传给模型的 prompt 只含：`session_id` + `task_hint[:150]` + `tools[:6]` + `turns` + `tool_calls`。**不传 user_text、asst_text、thinking_text**。LLM 看到的只有：

```
[520162bc] goal=测试？ tools=Agent,Bash,CronList,Edit,ExitPlanMode,Glob turns=42 calls=184
```

### 关键信息丢失链

```
JSONL 原文（30MB+）: thinking 完整推理链 + tool_use 参数 + 每轮对话全文
          ↓ truncate
SQLite turns 表（2KB/session）: user_text[2000] + asst_text[3000] + thinking_text[2000]
          ↓ 只取 task_hint[:200] + 统计数字
mine JSON（500B/session）: turns/tool_calls/tools/thinking_pct + LLM 一行摘要
          ↓ 只取 llm_summary + tools[:8] + turns/tool_calls
memory 文件（300B/session）: 一行摘要 + 元数据
```

每层丢的东西：
- **SQLite 层**就丢了 tool_use 的参数值（改了什么文件、传了什么参数、返回了什么）
- **mine 层**丢了每轮对话内容和 thinking 链
- **LLM 层**只看 task_hint 前 150 字符 + 工具名列表，看不到实际对话细节
- **memory 最终形态**：一行摘要，回答不了"具体改了什么"、"为什么这样改"
