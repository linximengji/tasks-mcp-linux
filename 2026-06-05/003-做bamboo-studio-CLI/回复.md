Bambu Studio CLI 可以作为离线切片引擎来生成 gcode，**但它本身不包含远程打印功能**（没有 `--send`、`--print` 等参数）。完整的远程打印流程需要 Bambu Studio CLI + p1s-mcp 组合：

```
STL → Bambu Studio CLI 切片 → gcode → p1s_upload → p1s_start_print
```

我验证了 Bambu Studio CLI 的可行性，但有几个注意事项：

**能做什么：**
- Bambu Studio CLI 支持 `--slice` 命令，可以 headless 切片
- 切片后的 gcode 可以用已有的 `p1s_upload` + `p1s_start_print` 发送到打印机
- Bambu Studio 已安装，不需要额外依赖

**需要解决的问题：**
- CLI 的配置文件兼容性有坑，需要把机器/工艺预设合并成无继承关系的扁平 JSON 才能被 CLI 加载（我验证过程中卡在这步）
- 不带图形界面的环境下切片能否稳定运行需要进一步测试

**建议方案：**

把 p1s-mcp 扩展一个工具（比如 `p1s_print_stl`），工作流：

```
用户输入 "打印 test_cube.stl"
  → MCP 调用 Bambu Studio CLI 切片（后台进程）
  → 生成 gcode 文件
  → p1s_upload 上传到 SD 卡
  → p1s_start_print 启动打印
```

要不要先把这个工具的脚手架搭起来？包括：
1. 从 Bambu Studio 导出完整的 P1S 0.4 喷嘴 + PLA 预设
2. 生成扁平 JSON 配置文件
3. 写 `p1s_slice` 工具（调用 Bambu Studio CLI 切片）
4. 写 `p1s_print_stl` 工具（一条命令完成切片+上传+打印）
