当前处理会话失效的机制，我梳理一下 `claude.ts` 中的三条路径：

**1. 自动重试 + 摘要注入**（第753-775行）

当 Claude CLI 返回 `No conversation found` / `Invalid session` / `Session not found` 等错误时：
- 删除失效 session 条目
- 如果该条目保存过 `sessionSummary`，将其注入到重试消息的头部
- 用新 session 重新发送消息（最多重试2次）

**2. summarize+reset**（第512-558行）

累计 token 超过 600K 时，主动生成摘要后清空 session，后续消息自动带摘要开场。但这条路需要事先有一个**正常工作**的 Claude 会话来生成摘要。

**3. 超过重试上限**（第760-762行）

两条路径都用完仍失败，直接抛错要求飞书端发"新会话"。

---

**现在的实际缺口**：`sessionSummary` 只在 summarize+reset 时写，而 reset 阈值是 600K token。如果 session 在达到阈值之前就崩了，重试时就没有摘要注入，会从零开始丢失上下文。

你想的是不是 —— 给每次对话在 token 还不多时（比如几轮交互后）就做一个轻量摘要备份，这样 session 意外失效时至少能带回去这个摘要？

或者你有别的方向想法？
