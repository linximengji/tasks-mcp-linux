好，想几个真正让手机端远程操作有质变的指令方向：

---

**1. `/fork` 分支会话** — 当前架构一个问题：一个 conversationId 只有一个 session。你想查个东西又不想打断主会话：

```
手机: /fork
Claude: 已创建分支会话 B，原会话 A 已保存
手机: 帮我看看 phone-archive.ts 的逻辑
Claude: 在分支 B 里读文件分析…
手机: /switch A    ← 切回主会话
手机: /switch B    ← 切回分支继续
手机: /forks       ← 列出所有分支
手机: /merge B     ← 把分支 B 的上下文合并回主会话
```

**实现代价**：低。`sessionMap` 已经是 key-value 结构，加一层别名映射即可。

---

**2. `/bg` 后台执行** — 有些任务不用等它跑完

```
手机: /bg 帮我重构 phone-archive.ts，改完发消息给我
Claude: ⏳ 已提交后台任务 #1，完成后通知你
// 手机端可以继续发其他消息，不影响
// 后台任务完成时，自动推送结果到飞书
手机: /tasks    ← 查后台任务状态
```

**实现代价**：中等。需要一个独立进程队列，跑完通过 feishu API 主动推消息。

---

**3. `/commit` / `/pr` git 操作** — 手机端最想做但做不了的事

```
手机: /commit "完成 phone archive 三分类"
Claude: ✅ committed 3 files, push? (y/n)
手机: /push
// 或者直接 /commit -p "xxx" 带 push
手机: /pr "Add conversation classifier"  ← 创建 PR
手机: /diff       ← 看当前改了哪些文件
手机: /log        ← 看最近提交
```

**实现代价**：低。纯 CLI 调用，不用改 Claude session。

---

**4. `/schedule` 定时任务** — 手机端设定期任务

```
手机: /schedule "每天早上 9 点跑半导体日报"
// 背后调用 CronCreate MCP
手机: /schedules     ← 查看已设定的定时任务
手机: /unschedule 1  ← 删除某条
```

**实现代价**：低。CronCreate 已经是可用的 deferred tool。

---

**5. `/scope` 作用域限定** — 让后续消息专注某个范围

```
手机: /scope src/core/
// 之后的每条消息都会加上"请在 src/core/ 范围内操作"
手机: 把 classifier 改为四分类
// → Claude 会在 scoped 范围内改
手机: /scope         ← 清除作用域
```

**实现代价**：低。在 message 前面 inject 一个上下文前缀就行。

---

**6. `/v <id>` 查看待办详情** — 串起现有的 phone-archive

```
手机: /v 2026-06-05/001-测试
// 读取 phone-tasks/2026-06-05/001-测试/消息.md 和 回复.md
手机: /tasks    ← 列出所有 pending 待办
手机: /done 1  ← 标记完成
```

**实现代价**：低。调 callClaude 发一个"读取 phone-tasks/xxx/消息.md"的指令即可。

---

**7. `/snapshot` 快照当前上下文** — 防止 session 意外丢失

```
手机: /snapshot "改完 classifier"
// 生成 summarize+reset 的效果，但不清空 session
// 这样 session 万一崩了，重连时能带摘要
手机: /snapshots     ← 查看快照历史
```

**实现代价**：低。复用已有的 `summarizeAndReset` 逻辑，最后不清空 sessionId 即可。

---

你觉得哪个方向有意思？我觉得 `/fork` + `/scope` 对手机端远程开发的体验提升最大，而 `/bg` 最实用（不用等）。
