从手机端的使用场景来看，当前指令只有清会话、重启、帮助，确实比较基础。我梳理几个有价值的维度：

---

**📋 待办查询** — `phone-tasks` 索引已经建好了，但手机端没法查
- `/tasks` 或 `/待办` — 列出所有 pending 的待办
- `/done <id>` 或 `/完成 <id>` — 标记待办已完成
- 和已有归档机制互补，闭环

**🔍 会话透明** — 手机看不到终端，不知道当前会话状态
- `/status` 或 `/状态` — 显示 session 活跃时长、累积 token 用量、是否已触发 summarize
- 手机端能感知"要不要发 `/new` 了"

**⏱ 手动压缩** — 感觉回复变慢/失忆时主动触发
- `/compact` 或 `/压缩` — 强制压缩当前 session
- 现在只有 token 到了阈值才自动触发，手机端主动控制更有用

**💾 快照** — 在重要节点保存上下文
- `/summary` 或 `/摘要` — 生成当前会话摘要并存入 sessionSummary，类似 summarize+reset 但不清空 session

---

**不会做**的：
- `/cancel` — 当前架构是同步 spawn claude CLI，等进程结束才能回调，没法从外部打断。架构上要大改（信号处理 + session 状态原子性），不值得
- `/restart` — 已经有了
