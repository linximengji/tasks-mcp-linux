根因找到了。问题在 `sanitize_for_deepseek` 中对 `thinking` block 的处理。


## 根因分析

**错误链路：**
1. 手机端（飞书）经过多轮对话后，Claude Code CLI 产生的 `assistant` 消息里包含 `thinking` 内容块（Claude 的 extended thinking）
2. 这些消息被 proxy 转发到 DeepSeek 时，调用 `sanitize_for_deepseek` 处理
3. `ALLOWED_BLOCK_TYPES` 包含了 `"thinking"`、`"redacted_thinking"`、`"tool_reference"` → 这些块**被保留**了下来
4. DeepSeek API 不认识这些 Anthropic 专有 block type → 返回 `400: "Unexpected item type in content"`
5. `/new` 清空 session → 历史消息中的 `thinking` 块全没了 → 暂时能用，但几轮对话后又会积攒起来

**关键代码** `proxy_lib/sanitize.py` 第 14-17 行：

```python
ALLOWED_BLOCK_TYPES = {
    "text", "tool_use", "tool_result",
    "thinking", "redacted_thinking", "tool_reference",  # ← 这仨不该出现在 DeepSeek 请求里
}
```

**对比：** `strip_thinking_blocks`（第 82 行）已经能正确干掉 thinking/redacted_thinking，但它只在 `anthropic` provider（qwen MAAS）时调用，DeepSeek 路由走的是 `sanitize_for_deepseek`，没调它。

## 为什么只出现在手机端

不是手机端专属，而是 **session 累积了足够多 thinking 块**后所有 DeepSeek 路由都会触发。手机端对话轮数多、走 DeepSeek 路由概率高，所以最先遇到。

要修吗？把 `thinking`/`redacted_thinking`/`tool_reference` 从 `ALLOWED_BLOCK_TYPES` 移除即可。
