可以。现在的分类器只输出 `task` / `reference`，改三分类就行：

| 输出 | 含义 | 动作 |
|---|---|---|
| `reference` | 闲聊/分析，无产出 | 只归档，不写 index.json |
| `task-pending` | 需要在终端/本地后续处理 | 写 `task-pending` |
| `completed` | Claude 已经通过工具做完了（修了bug/写了代码/生成了文件） | 写 `completed`，直接标记完成 |

分类器 prompt 里已经有这个判断逻辑了：

> 用户消息: 修一下这个 bug / 回复: bug 已修复 → reference（当前）
> 用户消息: 也加到终端待办事项里去 / 回复: 已经加上了 → task（当前）

改成三分类后，同一轮对话里 Claude 说"搞定了"就直接标 `completed`，一次到位。以后 Claude 在终端查待办列表时，已经完成的就不会再出现。
