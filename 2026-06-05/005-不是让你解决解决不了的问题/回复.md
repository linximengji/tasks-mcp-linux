好的，信息已经清楚了。OrcaSlicer 可以支持补全 P1S MCP 全功能。

**OrcaSlicer 相比 Bambu Studio CLI 的优势：**

| 维度 | Bambu Studio CLI | OrcaSlicer CLI |
|---|---|---|
| P1S 配置 | 自带 | 自带（0.2/0.4/0.6/0.8 全喷嘴） |
| CLI 稳定性 | 有兼容性 bug | v2.3.2 修复了 CLI segfault，可用 |
| 谱系 | 同一代码基 | 更活跃的 fork（14.5k stars） |
| 发送打印 | CLI 本身不支持 | CLI 也只负责切片，打印仍需 MQTT |

可下载文件：
- 安装版（119 MB）：`OrcaSlicer_Windows_Installer_V2.3.2.exe`
- 便携版（150 MB）：`OrcaSlicer_Windows_V2.3.2_portable.zip` —— **推荐用于 MCP 集成**

**对 P1S MCP 的补充方案：** 下载便携版放到 p1s-mcp 项目目录，用 CLI 完成切片，gcode 通过已有的 `p1s_upload` + `p1s_start_print`（或未来修复后）发送到打印机。

你先说要不要继续走这个路线？如果要的话：
1. 下载便携版解压验证 CLI 可用
2. 测试用 P1S 配置切片
3. 做 `p1s_slice` 工具补齐 MCP 功能链
