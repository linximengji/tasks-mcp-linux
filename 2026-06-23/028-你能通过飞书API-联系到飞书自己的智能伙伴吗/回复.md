**不能直接调用飞书内置的"智能伙伴"**。

飞书智能伙伴（Aily）目前**没有公开的 OpenAPI 可以直接以编程方式与其对话**。现有 API 都是面向用户自行创建的自定义智能体：

| 维度 | 说明 |
|------|------|
| 内置智能伙伴 | 飞书客户端自带的 AI 助手，**无 OpenAPI** |
| 自定义智能体 API | 你在 Aily 平台创建智能体后，可通过 `POST /open-apis/aily/v1/agents/:agent_id/chats` 调用 |
| 技能 API | 同步调用 Aily 应用中的技能，输入输出都是 JSON |
| 知识问答 API | SSE 流式 RAG 问答 |

如果你想要类似效果，有两个方案：

1. **在飞书 Aily 后台创建一个自定义智能体**，配置知识库和技能，然后我可以通过 OpenAPI 调用它
2. **用 `lark-im` skill 给飞书的机器人发消息**——我现有的飞书 IM 工具可以收发消息，如果智能伙伴被加到了某个群聊，理论上可以通过群聊消息间接交互（但通常智能伙伴是单独的聊天，不支持 API 代理对话）
