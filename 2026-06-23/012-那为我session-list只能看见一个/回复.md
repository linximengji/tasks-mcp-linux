之前的分析我搞混了两层东西，抱歉。正确的关系是：

## 三个层级，各司其职

| 位置 | 数量 | 用途 | 清除时机 |
|------|------|------|---------|
| **`~/.claude/sessions/*.json`** | **2 个** | **← `session list` 看的就是这个** | session 正常退出时自动删除 |
| `Temp/claude/D--ClaudeProjects/*/` | 401 个 | 工具执行缓存（tool results、任务输出） | Claude Code 自动管理，非正常退出时残留 |
| `session_knowledge_state.json` | 227 条 | hook 的"已处理标记" | 从不自动清理 |

## 你不是只能看到一个吗？没错

`session list` 只显示 `~/.claude/sessions/` 下的文件，目前就 **2 个**：
- 当前这个（pid 22076，运行中）
- 昨天那个（pid 9628，名为 "ops-dashboard-security"，因异常退出未清除，状态显示 `shell`）

正常退出后 `.json` 文件就删了，所以平时你 `session list` 往往只能看到 **当前会话自己**。

**Temp 的 401 个目录完全不影响 `session list`。** `--resume` 也只需要 sessionId 匹配时才会去 Temp 下面找缓存数据。

## 手机端 session

按这个逻辑，**手机端的 session 你也只能看到一个**：手机是遥控器，实际 claude session 跑在你这台机器上。Session 退出后 `.json` 删除，list 里就没了。不存在远程机器上的 session 被列出来的情况。
