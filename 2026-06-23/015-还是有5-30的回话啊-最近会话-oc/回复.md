查完了。结论很直接：

**`/session list` 里的 `oc_` 开头的会话存储在 Anthropic 服务器上**，不是本地文件。本地 `~/.claude/sessions/` 下的 `.json` 只是进程级 PID 映射，删了也没用。

而且 **Claude Code CLI 没有提供删除 session 的命令**——没有 `/session delete`，没有 `claude session delete`。你能做的只有 `/session list` 查看和 `/session switch <id>` 切换。

**那 05-30 的旧会话为什么还在？**服务器端没有自动清理机制（或者保留期很长），你看到的就是你所有历史会话。

**主动管理的限制：**
- 无法通过 CLI 删除 `oc_` 会话
- 本地删文件不影响列表
- 没有官方 API 端点在 Claude Code 中暴露

如果你在 claude.ai 网页端有 Code 页面，那里可能能看到这些会话，但也不确定是否有删除入口。

如果你非常想清理，可以考虑：
1. **开新进程时不恢复旧 session**——直接 `claude` 启动，不用 `--resume`
2. **检查 claude.ai 网页端**有没有对应用户会话管理

除此之外，目前只能接受它们留在列表里。
