## 2. 重复/冗余命令（合并建议）

### 手机待办 — 4个同义词
```
/tasks, 待办, 手机待办, 手机任务   → 全部做同一件事（读index.json列表）
```
**问题**：`待办` 太短，日常聊天中"待办一下这个"可能误触  
**建议**：砍到 2 个，保留 `/tasks` + `手机待办`

### 快速存档 vs 查看待办 — 名字打架
```
"加入手机待办" → 存档当前会话到待办
"/tasks"/"手机待办" → 查看已有待办列表
```
**问题**：都含"手机待办"，用户不知道哪个是"存"哪个是"看"  
**建议**：查看待办改成 `/tasks` 专用中文名 `任务列表`，存档保持 `加入手机待办`

### 远程指令 — 两套说法混用
```
开启远程, 打开远程   /   启动面板, 打开面板    → 同一个操作
关闭远程, 停止面板, 关闭面板                   → 同一个操作
```
**建议**：统一用"远程"，去掉"面板"变体

### Daemon 重启指令
```
/daemon restart, 重启daemon    → 正常
重启守护                        → "守护"不准确（守护进程≠守护）
```
**建议**：保留前两个，去掉 `重启守护`

---

## 3. 存档确认词范围太宽

存档确认监听以下词：
```
确认, 对, 是的, 是, 好, ok    （确认存档）
取消, 不算, 不要, 不是, 不, no  （取消存档）
```
**问题**：虽然只在 `_pendingArchive` 有状态时才拦截（不是全局），但突然说句"好"也会触发存档。  
**建议**：确认词缩到 `确认`/`好的`/`ok`，去掉 `对`/`是`/`好` 这种日常高频单字  
  取消词去掉 `不`/`不是`（太宽泛）
