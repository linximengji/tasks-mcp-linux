## `claude session list` 展示的是什么？

它展示的是 **Claude Code 本地持久化的所有历史会话**，每个目录对应一次 `claude` 交互式会话的完整上下文记录，以便你用 `--resume` 恢复之前的对话。

## 为什么还有 5 月 30 日的？

因为 **没有人清理它们**。会话数据存在两个地方：

1. **`%TEMP%\claude\D--ClaudeProjects\`** — 每个会话一个 UUID 目录，从 5 月 16 日累积到今天，共 **400 个**
2. **`~/.claude/session_knowledge_state.json`** — session-knowledge hook 的提取追踪记录，227 条

Windows 的 Temp 目录**不会自动清理子目录结构**，磁盘清理只删临时文件不删目录。所以所有会话目录原封不动。

## 要清掉吗？

如果你不需要 `--resume` 回到那么老的会话，可以清掉 6 月之前的：

```bash
# 清掉旧会话，保留最近 N 天
find /c/Users/34971/AppData/Local/Temp/claude/D--ClaudeProjects/ \
  -maxdepth 1 -type d -name "????????-*" -mtime +7 -exec rm -rf {} +
```

这会保留最近 7 天的会话（可继续 `--resume`），清掉更早的 350+ 个。
