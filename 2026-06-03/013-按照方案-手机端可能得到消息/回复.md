如果改 line 880 加上 `actualMessage`，手机端会收到类似：

```
处理消息时出错: claude exited with code 1. stderr: (empty)
发送的消息: 调查一下出错原因？
```

但 `actualMessage` 的实际内容取决于会话状态：

| 场景 | `actualMessage` 内容 | 手机端效果 |
|---|---|---|
| **私聊 / 新会话** | 纯用户输入（无模板拼接） | `发送的消息: 帮我写个脚本` → 简洁有用 |
| **群聊 resume** | 模板填充后的完整 context（含历史消息、群成员列表、发送者信息、消息原文） | `发送的消息: ### 上下文信息 ###\n👤 发送者: ...\n📱 群名: ...` → 几百到上千字，手机刷屏 |
| **群聊 /new 后** | 同上但无历史，仍含模板结构 | 比 resume 短，但仍有 500+ 字 |

**结论**：直接拼 `actualMessage` 不行——群聊场景会把整个模板砸到手机通知里，用户什么都看不清。

**实际可行的方案**：只拼用户的原始消息文本（不经过模板拼接）。对应代码里真正的用户输入就是 `options.message`（在 `index.ts` 传入的 `messageText`）。改为：

```
处理消息时出错: claude exited with code 1. stderr: (empty)。消息: 帮我写个脚本
```

手机端就始终只看到用户自己发的那句话 + crash 提示，不会刷屏。
