结论清楚了：

**根因：Claude CLI crash 时 stderr 始终为空，错误消息完全没有区分度。**

日志里今天出现了 6 次完全相同的错误：
```
处理消息时出错: claude exited with code 1. stderr: (empty)
```

| 时间 | 触发 |
|---|---|
| 14:38 | 手机消息 |
| 14:39 | 手机消息 |
| 14:56 | 手机消息 |
| 15:31 | 手机消息 |
| 15:35 | 手机消息 |
| 15:36 | 手机消息 |

**代码路径**（`claude.ts:880`）：

```ts
// 流式模式 crash（最常用）
reject(new Error(`claude exited with code ${code}. stderr: ${stderr || '(empty)'}`))
//                                            ^^^^^ stderr 始终空 → 毫无区分度
```

非流式版本（line 600）其实已经把 `stdout` 拼进去了，但流式版本没有——问题是手机端消息 99% 走流式路径。

**手机端看到的信息是**（`index.ts:278`）：
```ts
const errorText = `处理消息时出错: ${error.message}`
```

**丢失的关键信息**：
- 用户在问什么（`actualMessage` 已在作用域内）
- Claude 在做什么（`fullText` 累积回复、最近一次 tool call）
- 崩溃前输出的最后内容

要修的话改动很小——在 line 880 的 error message 里加上 `actualMessage` 前缀就足够区分了。要不要修？
