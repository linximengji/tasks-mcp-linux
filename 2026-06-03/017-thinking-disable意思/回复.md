`thinking` 是 DeepSeek/Anthropic 模型的"先想再说"机制——模型在输出可见文字之前，先产生内部推理 tokens（类似 o1 的 CoT）。这些 thinking tokens 计入 `max_tokens` 消耗。

对分类器的影响：

```
max_tokens=20, thinking 开启:
  thinking: "The user is asking about..."  ← 消耗 15 tokens
  text: "moderate"                         ← 只剩 5 tokens，第二个词来不及

max_tokens=100, thinking 关闭:
  text: "simple low"                       ← 全部用于文本输出
```

`"thinking": {"type": "disabled"}` 就是告诉 API 跳过推理阶段，直接输出文本。分类器不需要推理质量，只需要一个词，所以关掉 thinking 让 budget_est 能完整输出。
