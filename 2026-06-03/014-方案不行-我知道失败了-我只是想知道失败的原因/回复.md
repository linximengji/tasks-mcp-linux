Claude CLI 退出时，流式模式下已有的可收集信息：

| 信息 | 当前状态 | 含义 |
|---|---|---|
| `code` | 只用数字，未转 hex | `0xC0000005` = access violation（内存越界），`0xC0000142` = DLL init 失败 |
| `stderr` | 打印了但永远空 | stream-json 模式错误也走 stdout |
| `buffer` | 有，未用 | 最后一段未完成解析的 stdout，大概率含崩溃原因 |
| `lastEvent` | 有 `finalResult` 但未放进 error | 最后成功解析的 event 类型：`thinking` / `tool_use` / `tool_result` |

**方案**——error message 换成诊断信息，不拼用户消息：

```
处理消息时出错: claude exit code 0xC0000142（STATUS_DLL_INIT_FAILED）. 最后阶段: tool_use(Bash). stdout 末尾: {"type":"system","subtype":"error","message":"..."}
```

具体做法（`claude.ts:880` 附近）：
1. `code` 同时打数字和 hex（Windows SEH 崩溃码才有意义）
2. 拼上 `buffer` 最后 200 字符（大概率含 `"type":"system","subtype":"error"` 或 `"type":"result","is_error":true`）
3. 记录上一个成功解析的 event type→在 error message 里加 `最后阶段: tool_use(Bash)` 或 `最后阶段: thinking`
4. `stderr` 该拼还是拼（万一哪天有内容）

手机端效果会从毫无意义的 `claude exited with code 1. stderr: (empty)` 变成含崩溃码和上下文的有用信息。要动手吗？
