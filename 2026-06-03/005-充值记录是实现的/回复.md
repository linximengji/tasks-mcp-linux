三层：写入、查询、日报整合。

**写入** — `balance_query.py` 的 `record_recharge()`：

```python
# 写入 recharge_records.jsonl
{"date": "2026-06-03", "provider": "DeepSeek", "amount": 100.0, "currency": "CNY", "ts": "..."}
```

CLI 入口：

```
python balance_query.py recharge DeepSeek 100
python balance_query.py recharge "DashScope Token Plan" 200 2026-06-02
```

**查询** — `_recharges_between(provider, start, end)`：读 `recharge_records.jsonl`，按 `(start_date, end_date]` 半开区间汇总金额。半开是为了避免同一条记录在 D 日和 D+1 日各被计一次。

**日报整合** — `_daily_spend_for()`：

```python
spend = prev["balance"] - curr["balance"]       # 余额减少 = 消费
recharge = (curr.topped_up - prev.topped_up)     # DeepSeek 自动检测（API 返回）
         + (curr.granted - prev.granted)
manual = _recharges_between(provider, prev["date"], td)  # 手动记录
return spend + recharge + manual                  # 充值"倒扣"回来，避免负消费
```

三层修正叠加：DeepSeek 有 `topped_up`/`granted` 自动检测，DashScope/火山引擎等其他 provider 没有这类字段，全靠手动 `python balance_query.py recharge ...` 记录。
