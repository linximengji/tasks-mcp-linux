## 缺失的两个服务怎么处理

| 服务 | 剥离后的方案 |
|------|-------------|
| **Blender MCP** | 坐电脑前时本地开 Blender 直接用，不走 CC。或者需要时临时在本地启 tunnel 桥接，用完关掉 |
| **P1S MCP** | 打印场景本来就要求人在打印机旁（取件、换料），坐电脑前操作更合理。真需要远程时本地开 tunnel 桥接 |

这两个都是**物理世界需要你在场的操作**——Blender 看视口调模型、P1S 换料取件。远程操作它们的需求本身就不高频。剥离后不影响 95% 的日常使用。

## 唯一需要改写的东西

| 项目 | 动作 |
|------|------|
| CC hooks（~10 个 bash/PowerShell 脚本） | PowerShell → bash 重写，一次性工作 |
| settings.json 里的路径 | `C:\Users\34971\` → `/home/ubuntu/` |
| safekill | PowerShell 版 → bash 版（已有 `safekill.sh` 基础） |
| 项目代码里写死的 Windows 路径 | grep 全局搜一遍改掉 |
| Playwright | 云上装 `chromium` 依赖，headless 模式（已有经验，无坑） |

没有架构级改动，就是路径 + shell 方言的机械替换。
