## CC CLI 云上运行机制

CC 本质是交互式 REPL，确实不会「空转」——每次对话启动一个新进程，对话结束进程退出。所以 **24h 在线 ≠ CC 进程 24h 运行**，而是 **环境就绪 + 入口持久**。

```
ClaudeTalk (云) 持久监听飞书 Webhook
  └── 收到消息 → spawn CC 子进程
      └── CC 处理对话
          └── 需要 Blender → 通过 Cloudflare Tunnel 请求本地

本地 Windows:
  Cloudflared tunnel ← 接收云上 CC 的 MCP 请求
    └── 路由到 Blender MCP / P1S MCP (localhost:端口)
```

## MCP 连接方案（关键）

云上 CC 的 `.claude/settings.json` 里：

```json
{
  "mcpServers": {
    "blender-mcp": {
      // 指向本地 tunnel 暴露的端点
      "url": "https://blender-tunnel.example.com/sse"
    },
    "p1s-mcp": { ... },     // 同上
    "memory-mcp": {
      "command": "node",    // 云上本地启动
      "args": ["memory-mcp/dist/index.js"]
    },
    "model-proxy": {
      "url": "http://localhost:4000"  // 云上本地 proxy
    }
  }
}
```

## 方案对比

| 方案 | 月费 | 优点 | 缺点 |
|------|------|------|------|
| **A: 腾讯云轻量 2C4G 80GB** | ¥56 年付 | 性价比最高，8M带宽 | 不是自己控制 ECS 实例 |
| **B: 阿里云 ECS u1 2C4G** | ¥81 年付 | 网络稳定，EIP 可降配 | 3M 带宽稍小 |
| **C: 全本地 + 合盖不休眠** | ¥0 | 零成本 | 依赖笔记本 24h 在线 |
| **D: 只迁 proxy + memory-mcp**<br>到腾讯云轻量 + CC 仍本地 | ¥56 | 容灾 + 关键服务云化 | 本地关机时 CC 不可用 |

## 推荐方案：D 起步 → 逐步到 A

**阶段一（1天）**：买腾讯云轻量 2C4G 80GB，只迁 proxy + memory-mcp + ClaudeTalk。CC 仍本地跑，但 ClaudeTalk 改连云上 proxy。本地关机时飞书 bot 不中断。

**阶段二（2-3天）**：确认云上环境稳定后，把 CC 配置和 hooks 迁上去。本地只留硬件 MCP。

**阶段三（可选）**：发现 2C4G 不够 → 升配置也是轻量直接点几下的事。

要推进的话，我建议先出个 **阶段一的具体迁移计划**——腾讯云机器开通、环境初始化、proxy 和 memory-mcp 部署步骤。要吗？
