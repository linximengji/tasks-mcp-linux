## 白龙马 (Bailongma) 项目

**涛思数据 (TDengine) 团队** 开发的开源数据接入服务，Golang 写的。

核心功能就两个：**Prometheus** 和 **Telegraf** 采集的数据直接写入 TDengine 时序库，无需写代码，配置即用。

📍 GitHub: [github.com/taosdata/Bailongma](https://github.com/taosdata/Bailongma)

⚠️ **已归档（2022-01），功能被 [taosAdapter](https://github.com/taosdata/taosAdapter) 取代，不再维护。**

---

### 能不能适配你的场景？

直接说结论：**几乎不相关。**

看看你现在的技术栈和项目：

| 你在做的 | 白龙马做的 |
|---------|-----------|
| Claude Code / LLM proxy / Router | TDengine 时序数据写入 |
| RSS 日报管线 (semi-report) | Prometheus/Telegraf 数据桥接 |
| MCP 服务器 (P1S, Blender) | 物联网时序数据接入 |
| 飞书远程控制 / 手机同步 | 已归档、不再维护 |

你的项目里没有出现 **TDengine 时序数据库**、**Prometheus 监控**、**Telegraf 采集** 这三个要素中的任何一个，而且白龙马本身已经是废弃项目了。

如果你是在找 **数据管道 / ETL** 类的工具，你的场景（RSS 抓取→AI 分类→邮件发送）更适合直接用 Python/TS 脚本 + LLM 处理，不需要引入一个时序数据库的专用桥接工具。
