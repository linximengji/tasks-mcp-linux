## CC Hook 事件体系（共 27+ 种事件）

### 已覆盖的：2 种

`PreToolUse` + `PermissionRequest`，只覆盖了 **工具调用前** 这一个阶段。

### 事件分类全景

**工具执行相关**（你现在覆盖了前 2 个）：
- `PreToolUse` ← 已有
- `PermissionRequest` ← 已有
- `PostToolUse` — 工具执行成功后
- `PostToolUseFailure` — 工具执行失败后
- `PostToolBatch` — 一批并行工具全部完成

**用户输入相关**：
- `UserPromptSubmit` — 用户提交 prompt 时（可修改/拒绝）
- `UserPromptExpansion` — slash 命令展开时

**对话生命周期**：
- `Stop` — Claude 结束一轮回复（可强制继续）
- `StopFailure` — 因 API 错误结束
- `PreCompact` — 上下文压缩前（可拦截）
- `PostCompact` — 压缩后

**子 Agent 相关**：
- `SubagentStart` / `SubagentStop` — 子 Agent 启停

**会话/环境**：
- `SessionStart` / `SessionEnd`
- `ConfigChange` — 配置文件变更
- `CwdChanged` — 工作目录变更
- `FileChanged` — 监听文件变更
- `InstructionsLoaded` — CLAUDE.md 加载完成

**MCP/权限**：
- `Elicitation` — MCP 请求输入
- `PermissionDenied` — 自动模式拒绝工具时
- `WorktreeCreate` / `WorktreeRemove`

**任务系统**：
- `TaskCreated` / `TaskCompleted`

### Hook Handler 类型

| 类型 | 支持决策？ | 说明 |
|---|---|---|
| `command` | ✅ | stdio JSON 通信。你现在的两个就是这个 |
| `http` | ✅ | POST 到 URL |
| `mcp_tool` | ✅ | 调 MCP 工具 |
| `prompt` | ✅ | 单轮 LLM 评估 |
| `agent` | ✅ | 多轮子 Agent + tool |

## 推荐增加的 Hook

按优先级排序：

### 1. **PostToolUse** — 工具审计/日志

记录 Claude 每次实际执行了什么操作（PreToolUse 拦截过的不能算执行），写入审计日志文件。

适合场景：
- 追踪 Claude 写了哪些文件、改了哪些行
- 记录所有 shell 命令执行历史
- 排查"不知道什么时候改了什么东西"

```json
{
  "matcher": "Edit|Write|Bash",
  "hooks": [{
    "type": "command",
    "command": "python3 .../hooks/audit_logger.py",
    "timeout": 5
  }]
}
```

### 2. **Stop** — 完成通知

Claude 完成任务后自动推送到手机/飞书。

适合场景：
- 长时间任务（代码迁移、批量重构）完成后通知
- 配合 `phone-sync` skill 推送到飞书

```json
{
  "matcher": "*",
  "hooks": [{
    "type": "command",
    "command": "python3 .../hooks/notify_stop.py",
    "timeout": 5
  }]
}
```

### 3. **UserPromptSubmit** — Prompt 质量门

提交 prompt 前做一些自动化预处理：

- 检测是否忘了加 `/compact` 指令
- 自动追加上下文标记（当前分支、工作目录状态）
- 拦截空 prompt 或过于模糊的请求

### 4. **PreCompact** — 压缩前保存

上下文压缩是不可逆的。在想压缩前 dump 一份当前对话的摘要到日志文件，方便事后回溯。

### 5. **PostToolUseFailure 或 StopFailure** — 错误告警

工具失败或 API 错误时推送到飞书，早发现早处理。

### 6. **SessionStart** — 会话初始化

每次新会话启动时自动执行一些初始化操作：
- 更新日记/时间追踪
- 打印当前项目状态摘要

## 简单建议

你的两个现有 hook 都围绕 **命令行安全**。如果能再加 1-2 个，最实用的方向是：

1. **PostToolUse 审计** — 解决"它刚才干了什么"的信息缺失。记录所有 `Edit`/`Write`/`Bash`
2. **Stop 通知** — 飞书推送"任务完成"消息，配合手机场景
