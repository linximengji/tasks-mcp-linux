核心思路是两个 hook 配合工作：

**PostToolUseFailure** — 记录每次工具失败的「签名」到文件
**PreToolUse** — 每次工具执行前检查：这个签名最近失败过吗？

### 签名计算

不是简单比 command 字符串，而是提取关键特征：

| 工具 | 签名依据 |
|---|---|
| Edit | `file_path` + `old_string`（对同一段代码反复改 = 同一签名） |
| Write | `file_path`（同一文件反复写 = 同一签名） |
| Bash/PS | `command`（同一命令反复跑 = 同一签名） |

用 SHA256 前 16 位做短签名，存到 `~/.claude/audit/failure-memory.jsonl`。

### 两级响应

**第 1 级 — 警告（retry 1-2 次）**：PreToolUse 检测到签名匹配，输出 `systemMessage` 注入 CC 上下文，提醒"此操作 30 分钟内失败过 N 次，上次错误：XXX"。CC 看到后会自己调整。

**第 2 级 — 阻断（retry ≥ 3 次）**：PreToolUse exit non-zero，直接拒绝执行。CC 收到 tool call failure，被迫换方案。

### 新增文件

- `~/.claude/hooks/post_failure_recorder.py` — PostToolUseFailure 钩子，记录失败
- `~/.claude/hooks/pre_tool_retry_guard.py` — PreToolUse 钩子，检测+警告/阻断
- `~/.claude/settings.json` — 加两条 hook 配置

### 为什么比「人工观察」有效

CC 的循环重试本质是**上下文污染**——前几次失败的输出在上下文里，但模型注意力衰减，越往后越忽略早期错误。`systemMessage` 注入会直接打断注意力，比自然对话里的错误信息有效得多。第 3 次阻断是硬兜底。
