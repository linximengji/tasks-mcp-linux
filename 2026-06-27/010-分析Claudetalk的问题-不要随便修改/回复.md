## 完整分析

### 问题 ①：Claude CLI 子进程崩溃（0xC0000142 — STATUS_DLL_INIT_FAILED）

这是**用户手机收到通知的直接原因**。exit-marker 准确记录了：

```
uid=4764，uptime=2369s（39分钟正常服务）
→ 用户发消息 → spawn('claude', args, {shell: true})
→ 子进程瞬间崩，exit code 3221225794 (= 0xC0000142, STATUS_DLL_INIT_FAILED)
→ stderr: (empty)——DLL 加载失败没来得及输出任何错误
```

**今天截至 15:08 已发生 75 次**，且高频集中在两个时段：

| 时段 | 模式 |
|------|------|
| 14:01:49 | exit code **null**（进程被外部 kill）|
| 14:37~14:38 | exit code **3221225794** × 连续 6 次（同一用户连续重试）|

`shell: true`（claude.ts:1043）在 Windows 下启动 claude CLI 时可能引入了额外的 DLL 依赖，加上 proxy 环境下 `claude` 命令本身的 PATH 解析路径变化导致加载失败。

### 问题 ②：3 个 claudetalk 实例同时运行（最关键）

```
PID 22016  00:21   [daemon] node --unhandled-rejections=warn cli.js --profile default
PID 4764   14:28   [C:\Program Files\nodejs\node.exe] cli.js --profile default   ← 无 --unhandled-rejections
PID 6552   14:29   [daemon] node --unhandled-rejections=warn cli.js --profile default
```

**daemon 每次重启 → spawn_claudetalk() 写新 PID** → 新实例启动，但旧实例没被杀 → **旧实例成了孤儿进程**，继续监听飞书消息。当用户发消息，哪个实例先收到就谁处理。旧实例的 Claude CLI 调用走到 proxy 链路时，路由状态已经不匹配（proxy 可能被 daemon 重启重建了），于是产生 `STATUS_DLL_INIT_FAILED`。

这也是 75 次高频崩溃的根本原因——不是同一个进程反复崩，是 **3 个进程各崩各的**。

### 问题 ③：进程实际没退出，但用户收到"异常退出"

`process_manager.py:193` spawn claudetalk 时加了 `--unhandled-rejections=warn`：
```python
[NODE, "--unhandled-rejections=warn", CLAUDETALK_CLI, "--profile", "default"]
```
这个 flag 让 Node.js **不杀死进程**，只打印警告。但 `index.ts:464` 的 `_trySendCrashNotification` 每次 unhandled rejection 都会发送：

```
[Crash] claudetalk 异常退出
原因: unhandledRejection: claude exited with code 3221225794. stderr: (empty)
时间: 2026/6/27 14:37:51
运行: 550s
```

**claudetalk 进程没退出**（`--unhandled-rejections=warn` 阻止了 exit），但通知写的 `[Crash] claudetalk 异常退出` 是误导。

### 问题链总结

```
daemon 因 OTLP 卡死 → TS 杀 daemon → daemon 重开 →
  spawn_claudetalk() 写新 PID → 新实例启动，旧实例不死 →
    3 个 claudetalk 并存，共用同一飞书 WebSocket →
      用户发消息 → 随机路由到旧实例 → spawn 'claude' CLI →
        shell:true 下 DLL 加载失败 → exit code 3221225794 →
          claudetalk 重试 2 次仍失败 → unhandledRejection →
            _trySendCrashNotification("[Crash] claudetalk 异常退出")
```

飞书收到的是这条 `_trySendCrashNotification` 消息，内容是 **claude CLI 进程崩了**但标题写的是**claudetalk异常退出**。

### 需要修什么

1. **singleton 机制失效** — daemon 重启后 spawn 新实例但不杀旧实例。根源：daemon 预写 PID 文件（process_manager.py:199-200）导致新实例的 singleton check 总是"通过"（`oldPid === process.pid`）
2. **`_trySendCrashNotification` 措辞误导** — unhandledRejection 时进程没退出
3. **`shell: true`** — Windows 下 spawn claude CLI 额外引入 DLL 依赖风险
