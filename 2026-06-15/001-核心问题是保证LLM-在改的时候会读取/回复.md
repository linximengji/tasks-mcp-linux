## 问题本质

记忆是 **pull** 机制——需要 LLM 主动 `memory_recall`。但 LLM 不会在每次 `Edit` 前"想起来"去查一下。

要解决的是**把 pull 变成 push**——让依赖信息在 LLM 动手时已经出现在上下文里。

## 可行方案：CLAUDE.md 内嵌文件速查表 + 规则约束

完整的功能切面映射存 memory（详细，按需检索），但加一个**精简反向索引**到 CLAUDE.md，让它始终在上下文里：

```markdown
## 文件功能切面速查（编辑前必查）
# 格式：文件路径 → 所属功能切面。完整 touchpoints 用 memory_recall 查
- src/core/classifier.ts → message_ingress, card_action_handling
- src/core/claude.ts → message_ingress  
- src/core/session.ts → message_ingress
- src/channels/feishu/index_feishu.ts → message_ingress, card_action_handling
- src/mcp-server.ts → card_action_handling, mcp_tools
- .env → 全局配置（所有切面）
```

40 个文件的项目大概 40 行，~1.5K 字符。CLAUDE.md 始终加载，**不看也得看**。

再加一条硬约束规则：

> 编辑文件前，根据速查表确认所属功能切面，必要时 `memory_recall namespace=<project>` 查完整 touchpoints。新增文件需同步更新速查表。

## 为什么这比纯记忆方案可靠

| | 纯 memory | CLAUDE.md 速查表 |
|---|---|---|
| 触发方式 | LLM 主动 recall | 每次会话自动加载 |
| 遗漏概率 | 高（忘了查） | 低（就在眼前） |
| 维护成本 | 改功能切面时更新 | 新增/删除文件时更新 |
| 体积 | 3-4K 字符 | ~1.5K 字符 |

## 关键设计决策

速查表只存**文件→切面**的映射（最小信息），不存完整 touchpoints。需要完整依赖链时，看到切面名后 `memory_recall` 查详细定义。这样 CLAUDE.md 不会膨胀。

记忆里存的是**切面定义**（touchpoints、config、consumers），CLAUDE.md 存的是**文件归属**（反向索引）。两者互补，各司其职。
