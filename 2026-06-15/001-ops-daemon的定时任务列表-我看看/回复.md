以下是 ops-daemon 的完整定时任务清单：

---

### 一、Scheduler 注册的 cron 任务（`Scheduler` 类，按 cron 触发）

| 任务 | cron | 频率 | 功能 |
|------|------|------|------|
| **baseline** | `0 * * * *` | 每小时 | 记录 CPU、内存、磁盘基准数据 |
| **daily_report** | `0 8 * * *` | 每天 08:00 | 运行日报告 `run_daily_report()` |
| **memory_audit** | `17 8 * * *` | 每天 08:17 | 运行 memory-mcp 日常审计（清理过期/低质记忆） |
| **semi_report** | `2 8 * * *` | ⚠️ 已配置未注册 | 配置存在但 `main.py` 里没有 `scheduler.add_task`，不会触发 |
| **token_daily** | `1 0 * * *` | ⚠️ 已配置未注册 | 同上，配置存在但未加入 scheduler |

semi_report 和 token_daily 在 `config.yaml` 里配了 cron，但 `main.py:282-293` 只注册了 baseline / daily_report / memory_audit 三个，剩下两个是死配置。

---

### 二、主循环周期检查（`daemon.run()` 每 60s 执行一轮）

| 检查 | 功能 |
|------|------|
| **proxy** | 检测 port 4000/4001 是否活着，不通则自动 spawn |
| **claudetalk** | 检测 port 9877 |
| **cloudflared** | 隧道健康 + 持续 degraded 超 600s 自动停服务 |
| **services** | 检查 MCP 子进程（p1s/blender/image-gen/pdf/drawio/openscad） |
| **system** | 磁盘/CPU/内存告警阈值 |
| **ssl** | 证书到期检查（30 天预警、7 天严重） |
| **logs** | 日志文件时效（48h） |
| **zombies** | MCP 僵尸进程收割（超 120min 的孤立进程） |
| **scheduled_tasks** | Windows Task Scheduler 状态审计 |
| **memory_health** | memory-mcp 健康检查（每 7 天运行一轮） |
| **semi_report** | 今日半导体日报是否已生成（超 9 点检查） |

---

### 三、修复钩子（检查失败后自动触发）

proxy、cloudflared 注册了 `REPAIR_REGISTRY` handler，检测失败后自动尝试恢复，最多重试 2 次、间隔 120s。
