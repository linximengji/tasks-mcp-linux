SessionStart hook 对 memory tree 的加载范围只有 **一个文件、一层深度**：

**来源**：`D:\ClaudeProjects\.claude\memory\MEMORY.md`（单文件、16-18 行）

**解析逻辑**（`_parse_index` at `session_start.py:19-30`）：只匹配 `- **{namespace}** — {description} (segments: {...})` 这一种格式的正则，提取 namespace 名、一行描述、segments 列表。

**输出到上下文中**：只是一个**扁平索引**，每 namespace 一行，例如：
```
- **claudetalk** (当前项目) — Node.js 飞书群聊... [_index, architecture, changelog, pitfalls]
- **(global)** — 全局 MCP 配置根... [_index, architecture, changelog, pitfalls, service-catalog]
- **memory-mcp** — 统一记忆管理系统... [_index, architecture, changelog, pitfalls]
...
```

**不加载的内容**：
- 各 namespace 目录下的具体 segment 文件（`_index.md`、`architecture.md` 等）— 不读
- segment 内的具体正文内容 — 不读
- 目录树结构遍历 — 不做

**后续访问方式**：只有通过系统提示提示的 `memory_list namespace=<ns>` 和 `memory_get <ns>/<segment>` 命令按需读取。即 lazy loading，启动时不预加载深层内容。
