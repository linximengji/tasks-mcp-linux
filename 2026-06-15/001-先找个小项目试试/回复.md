### 这个架构段的"隐性价值"点

一般的重构工具/静态分析能告诉你 import 链，但存不下来这些：

| 条目 | 为什么 import 看不到 |
|------|-------------------|
| `STORAGE_KEY("upstreamUrl")` 被 3 个文件读写 | 跨 chrome.storage.sync，不是 import |
| background ↔ content/trace.js 的 `TRACE_EVENT` 协议格式 | 通过 `chrome.runtime.onMessage` 通信 |
| 新增 action 必须同步更新 `handleCommand` 的 switch-case | 纯字符串映射 |
| ops-daemon 是外部消费者 | 完全不在代码里 |
| `debugger version "1.3"` 和 error 匹配字符串 | CDP 协议常量 |

### 体积评估

8 个切面 + 2 个表格 + 1 个外部消费者引用，总计 ~2.1K 字符。放在 memory-mcp，`memory_get` 一次读的时间几乎不计。
