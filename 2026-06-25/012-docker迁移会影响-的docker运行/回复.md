## Docker WSL 数据迁移分析

**当前 Docker 状态：**
- 4 个容器在运行：searxng、pact-broker、jaeger、presenton
- 7 个镜像，~4 GB（虚拟大小）
- 磁盘数据：`C:\Users\34971\AppData\Local\Docker\wsl\disk\docker_data.vhdx` = **6.62 GB**
- Docker 后端是 WSL 2，数据全在 vhdx 里

### 迁移方案：WSL export/import

```
1.  docker image prune -a        # 先瘦身，清理未用镜像
2.  stop Docker Desktop            # 停止 Docker
3.  wsl --shutdown                 # 关闭所有 WSL 实例
4.  wsl --export docker-desktop-data D:\docker\backup\docker-data.tar
    # ↑ 导出整个数据盘（含镜像/容器/卷）
5.  wsl --unregister docker-desktop-data
6.  wsl --import docker-desktop-data D:\docker\data\ D:\docker\backup\docker-data.tar --version 2
    # ↑ vhdx 现在位于 D:\docker\data\
7.  启动 Docker Desktop 验证
```

### 影响

| 项目 | 影响 |
|------|------|
| **停机时间** | 容器停 ~5-10 分钟（export 大型 tar 较慢） |
| **数据安全** | 镜像、容器、卷**全部保留**，只是换个位置 |
| **风险** | export/import 过程中若断电/崩溃，tar 文件可能损坏 |
| **回退方案** | 有任何问题，原地用 tar 反向 import 回 C 盘即可 |

### 关键是——**真的需要迁吗？**

当前 Docker 数据 6.6 GB，在 C 盘占比并不大。如果你 C 盘主要问题是 **npm/pip/uv 那 8.5 GB 已经回收了**，加上 **ms-playwright (1.3G) + puppeteer (1.3G) + Temp 清理 (800M)**，总共能释放 ~12 GB，C 盘就从 74% 降到 ~68%。可能没必要冒 Docker 迁移的险。
