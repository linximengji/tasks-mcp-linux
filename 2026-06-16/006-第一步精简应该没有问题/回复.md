## 计划概要

### Phase 1 — 简化提取 hook

**现状问题：** 每次 hook 触发都生成长 prompt + 调 LLM + import _storage + 写 audit，大部分低价值会话白花钱。

**改后流程：** `hook → parse_turns → sample_turns → 规则判断（≥3 轮 + 有工具调用或 anchor）` → 写 staging JSONL。**不调 LLM，不 import _storage，不发飞书。**

**删掉：** `consume_queue()` / `fmt_prompt()` / `call_llm()` / `parse_llm()` / `store_mem()` / `_process_one()` — 整段去掉约 300 行。

**影响文件：** 仅 `session_knowledge.py`

### Phase 3 — 清除废弃物

- `_storage.py`：去掉 `_ensure_audit_record()` / `_strip_audit()` / `_extract_audit()` / `AUDIT_RECORD_RE` / `memory_review()` / `_run_audit()` 
- `daily_audit.py`：保留代码但不从 ops-daemon 调度（`config.yaml` 去掉 `tasks.memory_audit`，`main.py` 去掉 `run_memory_audit()`）
- 一次性脚本清理现存 200+ 条记忆中的 audit-record 注释块

### 不涉及

- `run_knowledge_hook.ps1` / `settings.json` 不变
- Phase 2 curator 排期待定

### 验证

1. 简短对话后确认 `~/.claude/knowledge_staging/` 写入了 staging 文件
2. 确认 `knowledge_queue.jsonl` 不再生成
3. 确认 SessionStart/SessionEnd hook 不报错
4. 确认 `_storage.memory_recall()` / `memory_get()` 正常
5. 确认现有记忆 frontmatter 正常
