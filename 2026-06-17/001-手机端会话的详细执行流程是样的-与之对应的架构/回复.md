## 手机端私聊会话完整执行流程

### 一、整体架构分层

```
飞书手机端 → 飞书开放平台 → WSClient 长连接 → FeishuClient → index.ts 消息处理器 → claude.ts CLI 调用层 → Claude Code CLI
                                       ↓
                               phone-archive.ts 归档管线
```

### 二、详细流程逐段拆解

#### 0. 启动阶段 (index.ts: `startBot`)

1. `startMcpServer(workDir)` — 启动 MCP SSE Server (port 9877)
2. `loadConfig()` — 加载 `.claudetalk.json`（含飞书凭据、systemPrompt、subagentEnabled 等）
3. `createChannel('feishu', ...)` → `FeishuClient` 实例化
4. 注册 `channel.onMessage()` 统一消息处理器 → **这是核心路由入口**
5. `channel.start()` 进入事件循环

#### 1. 连接层 (index_feishu.ts)

**双通道接收：**

```
通道 A: WSClient 长连接（主力）
  飞书 SDK(larksuiteoapi/node-sdk) → WebSocket → 自动认证/心跳/重连
  → eventDispatcher.register:
      - 'im.message.receive_v1' → handleMessageEventAsync()
      - 'card.action.trigger'   → handleCardAction()
      - reaction created/deleted → 空处理器（防告警）

通道 B: Peer Message 轮询（辅力）
  setInterval(5000ms) → 读 bot_{name}.json 文件
  → 处理 createdAt+10s 已到期的消息
  → 同走 channelMessageHandler 流程
```

#### 2. 消息接收与预处理 (index_feishu.ts `handleMessageEventAsync`)

```
WS 收到事件 → 去重(event_id/24h TTL) → checkAndPrepare():
  1. 群聊策略检查：disabled / at_only（默认） / allowlist
  2. 私聊策略检查：allowlist 过滤
  3. 消息类型检查：只支持 text/image/post/file/audio
  4. parseFeishuMessage():
     - text:  JSON 解析文字
     - image: 下载到 .claudetalk/feishu/images/，返回路径
     - post:  提取文字 + 下载嵌入图片
     - file:  下载到 .claudetalk/feishu/files/，返回路径+文件名
     - audio: 飞书 ASR API → 语音转文字（就是你刚才遇到的识别失败场景）
  5. 去掉 @机器人 / @_user_N 占位符
  6. 纯图片消息 → 缓存到 pendingImages Map，回复 "已收到"
  7. buildFinalMessage(): 合并缓存图片路径到文字消息末尾
                          （最多拼接 3 个附件，超出截断）

  → 调用 channel.onMessage 注册的处理器（index.ts 中的大闭包）
```

#### 3. 命令路由层 (index.ts `channel.onMessage`)

```
收到 finalMessageText → strippedMessage → command（小写）

├─ 内置指令（纯逻辑，不走 LLM）:
│  RESET_COMMANDS → clearSession() 删 .claudetalk-sessions.json 条目
│  HELP_COMMANDS  → 回复帮助文本
│  RESTART_COMMANDS → 1s 后 process.exit(0)（仅私聊）
│  STATUS_COMMANDS → 健康检查 Proxy/Dashboard/Tunnel/daemon
│  TASKS_COMMANDS  → 读 phone-tasks/index.json
│  MODEL_COMMANDS  → 检查 /v1/models 可达性
│  UPTIME_COMMANDS → process.uptime()
│  LOG_COMMANDS    → 读今日日志尾 20 行
│  DAEMON_RESTART_COMMANDS → 写 .stop 文件触发 ops-daemon 重启
│  开启远程/关闭远程 → cloudflared tunnel + ops-dashboard 启停
│
└─ 默认路由 → callClaudeStreaming()（首选） / callClaude()
```

#### 4. Session 管理层 (claude.ts)

```
┌─ Session 持久化 ──────────────────────────────────────────┐
│  .claudetalk-sessions.json                                 │
│  key = conversationId\x00workDir\x00profile\x00channel     │
│  value = { sessionId, lastActiveAt, cumulatedInputTokens,  │
│            sessionSummary, subagentEnabled, ... }           │
│  sessionMapCache: 按 workDir 缓存，避免每次读文件           │
└───────────────────────────────────────────────────────────┘

┌─ 串行化队列 ──────────────────────────────────────────────┐
│  sessionQueues Map<sessionKey, Promise>                    │
│  每来一条消息排队等前一条完成（防止并发 fork session）       │
└───────────────────────────────────────────────────────────┘

┌─ 三级 Token 阈值管理 ─────────────────────────────────────┐
│  200K: 回复完成后异步 spawn /compact                       │
│  400K: 请求前同步等待 /compact 完成                        │
│  600K: summarize + reset（新 session + 摘要注入）          │
│                                                            │
│  修复历史（changelog #8）：第一次 reset 成功后              │
│  sessionSummary 非空 → 守卫永远 false → 一直不触发 reset   │
│  → session 增长到 855 万 token → OOM 崩溃                  │
│  修复：删除 !existingEntry.sessionSummary 守卫条件          │
└───────────────────────────────────────────────────────────┘

┌─ 重试策略 ────────────────────────────────────────────────┐
│  Session 无效（No conversation found）：最多 2 次重试       │
│  普通错误（crash/timeout）：最多 2 次重试                   │
│  配置变更（subagentEnabled 开关）：清除旧 session，递归重试 │
│  Timeout: 120s → SIGTERM → 3s → SIGKILL                    │
└───────────────────────────────────────────────────────────┘
```

#### 5. Claude CLI 调用 (claude.ts)

**流式 (callClaudeStreaming) 路径 — 手机端主路径：**

```
buildClaudeArgs() → 参数组装:
  - 新会话: claude -p --output-format stream-json --verbose --dangerously-skip-permissions
            [+ --agents (subagentEnabled) 或 --append-system-prompt (非 subagent)]
  - 续会话: 同上 + --resume <sessionId>
  - 摘要注入: [Previous conversation summary: ...]\n\n{message}

→ spawn('claude', args) pipe stdin/stdout
→ stdin.write(actualMessage) + stdin.end()

→ stdout 逐行 JSON 解析 → onEvent 回调:
  - {type:'assistant', content[{type:'thinking'}]}  → '🧠 思考中'
  - {type:'assistant', content[{type:'tool_use'}]}  → '第N步: 工具名'
  - {type:'user', content[{type:'tool_result'}]}    → summarizeStep() 推一句话摘要
  - {type:'assistant', content[{type:'text'}]}       → '回复内容'
  - {type:'result'}                                  → 完成

→ editMessage (PUT /im/v1/messages/:id) 实时更新飞书消息内容
  限频 800ms，超编辑上限(20次) 后发新消息

→ session 持久化（cumulatedInputTokens 递增）

→ token 日志追加到 .claudetalk/token_usage.jsonl
```

**非流式 (callClaude) 路径 — 降级：**

```
同参数组装，但 output-format=json
→ await child exit → 解析 JSON 响应 → 获取 result + sessionId
→ session 持久化 + token 日志
```

#### 6. 后处理管线 (phone-archive.ts)

```
archiveConversation() — 仅私聊触发：

1. makeSlug(message) → 去掉填充词/URL/@，截取前 25 字符
2. nextSeq() → 按日期文件夹自增序号 (001, 002...)
3. 创建目录: phone-tasks/{date}/{seq}-{slug}/
4. 写入 消息.md + 回复.md
5. 图片附件硬链接到 attachments/
6. classifyConversation() 三步分类:
   - 本地关键词预检: PENDING_RE / COMPLETED_RE
   - LLM 调用: deepseek-v4-flash（带 1 次网络重试）
   - 降级 fallback: toolNames > 2 → completed
7. 分类结果写入 index.json:
   - completed → status: completed，不回飞书
   - task-pending → status: pending，飞书卡片 + "完成"按钮
   - reference → 不入 index，仅写 INDEX.md 日志
8. INDEX.md 追加一行归档摘要
```

#### 7. 卡片回调 (index_feishu.ts `handleCardAction`)

```
card.action.trigger → 3s 内必须响应:

├─ mark-task-done:
│  1. 同步：修改 index.json status → 'completed'
│  2. 异步：PATCH /im/v1/messages/:id 替换卡片为绿色"已完成"样式
│  返回 { toast: "✅ 已标记完成" }
│
├─ execute:
│  异步 spawn callClaude() 处理 → 完成后回复飞书
│  返回 { toast: "正在执行..." }
│
├─ toast: 显示提示
├─ dismiss: 忽略
└─ default: "未知操作"
```

### 四、关键设计决策

| 决策 | 理由 |
|---|---|
| **流式优先，非流式降级** | 飞书消息编辑 API (PUT) 支持最多 20 次更新，实现实时进度推送。群聊不支持编辑 → 自动走非流式 |
| **三级 Token 阈值** | 防止单 session 无限增长导致 OOM（历史教训：855 万 token crash）|
| **sessionQueues 串行化** | Claude CLI 是单进程线性 session，并发 fork 会导致 session 分裂 |
| **关键词预检 + LLM 分类双保险** | LLM 分类器超时/失败时有本地规则兜底，不影响归档主线 |
| **图片多消息缓存** | 先收图片，等文字指令到后一并发送，减少用户操作步骤 |
| **Peer Message 轮询** | 实现双机器人同群协作：@其他机器人 → 文件写 → 目标机器人轮询消费 → 回复 |
